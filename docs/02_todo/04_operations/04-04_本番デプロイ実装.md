# 04-04 本番デプロイ実装

## 概要
本番環境にデプロイし、顧客がVercel URLでサービスを利用できるようにする。FastAPIサーバーとNext.jsアプリをそれぞれ適切なプラットフォームにデプロイする。

## 詳細タスク

### フェーズ1: FastAPIサーバーのデプロイ

- [ ] FastAPIサーバーのデプロイプラットフォームを選択する
  - Railway、Google Cloud Run、Renderのいずれかを選択する
  - 各プラットフォームの特徴を確認する
  - コストと運用のしやすさを考慮する

- [ ] RailwayでFastAPIサーバーをデプロイする（オプションA）
  - Railwayアカウントを作成する
  - 新しいプロジェクトを作成する
  - GitHubリポジトリを接続する
  - `api`ディレクトリをルートとして設定する
  - 環境変数を設定する（必要な場合）
  - デプロイを実行する
  - デプロイURLを取得する（例: `https://numbers-ai-api.railway.app`）

- [ ] Google Cloud RunでFastAPIサーバーをデプロイする（オプションB）
  - Google Cloudプロジェクトを作成する
  - Cloud Run APIを有効化する
  - `api/Dockerfile`を使用してコンテナをビルドする
  - Container Registryにプッシュする
  - Cloud Runにデプロイする
  - URLを取得する

- [ ] RenderでFastAPIサーバーをデプロイする（オプションC）
  - Renderアカウントを作成する
  - 新しいWebサービスを作成する
  - GitHubリポジトリを接続する
  - ビルドコマンドを設定する（`pip install -r requirements.txt`）
  - 起動コマンドを設定する（`uvicorn main:app --host 0.0.0.0 --port $PORT`）
  - デプロイを実行する
  - URLを取得する

- [ ] FastAPIサーバーの動作確認をする
  - デプロイURLにアクセスしてヘルスチェックを確認する（`/health`エンドポイント）
  - APIドキュメントにアクセスできることを確認する（`/docs`エンドポイント）
  - ログを確認してエラーがないことを確認する

### フェーズ2: CORS設定の更新

- [ ] 環境変数から許可オリジンを読み込むように変更する
  - `api/main.py`のCORS設定を確認する
  - 環境変数`ALLOWED_ORIGINS`を追加する
  - デフォルト値として開発環境のURLを設定する
  - 本番環境ではVercel URLを環境変数で指定する

- [ ] CORS設定のコードを更新する
  - `allow_origins=["*"]`を環境変数ベースに変更する
  - 開発環境と本番環境の両方に対応する
  - セキュリティを強化する

- [ ] 変更をコミット・プッシュする
  - 変更をコミットする
  - GitHubにプッシュする
  - FastAPIサーバーを再デプロイする

### フェーズ3: Next.jsアプリのVercelデプロイ

- [ ] Vercelアカウントを作成・ログインする
  - Vercelダッシュボードにアクセスする
  - GitHubアカウントでログインする

- [ ] Vercelプロジェクトを作成する
  - 「Add New...」→「Project」をクリックする
  - GitHubリポジトリ `numbers-ai` を選択する
  - プロジェクト名を確認する

- [ ] ビルド設定を確認する
  - `vercel.json`が正しく設定されていることを確認する
  - Framework: Next.jsが自動検出されることを確認する
  - Build Command: `pnpm build`を確認する
  - Region: `hnd1` (東京)を確認する

- [ ] 環境変数を設定する
  - プロジェクト設定 → 「Environment Variables」を開く
  - `FASTAPI_URL`を追加する（フェーズ1で取得したFastAPIサーバーのURL）
  - Environment: Production, Preview, Development すべてにチェックする
  - 「Save」をクリックする

- [ ] 初回デプロイを実行する
  - 「Deploy」ボタンをクリックする
  - デプロイが完了するまで待機する（数分）
  - デプロイURLを取得する（例: `https://numbers-ai.vercel.app`）

- [ ] FastAPIサーバーのCORS設定を更新する
  - フェーズ2で更新したCORS設定にVercel URLを追加する
  - 環境変数`ALLOWED_ORIGINS`にVercel URLを含める
  - FastAPIサーバーを再デプロイする

- [ ] Vercelを再デプロイする
  - 環境変数更新後、再デプロイを実行する
  - デプロイが成功することを確認する

### フェーズ4: 動作確認

- [ ] デプロイされたVercel URLにアクセスする
  - ブラウザでVercel URLを開く
  - ページが正常に表示されることを確認する

- [ ] 予測機能が正常に動作するか確認する
  - 回号を入力して予測を実行する
  - リハーサル数字を入力して予測を実行する
  - 予測結果が正しく表示されることを確認する

- [ ] ブラウザの開発者ツールでエラーを確認する
  - 開発者ツール（F12）を開く
  - Consoleタブでエラーがないことを確認する
  - NetworkタブでAPIリクエストが成功していることを確認する

- [ ] パフォーマンスを確認する
  - 初期ローディング時間を測定する（目標3秒以内）
  - AI予測応答時間を測定する（目標5秒以内）
  - Lighthouseスコアを確認する（目標90+）

- [ ] モバイルデバイスでの動作確認をする
  - スマートフォンでアクセスする
  - 画面サイズが正しく表示されることを確認する
  - タッチ操作が正しく動作することを確認する

- [ ] ブラウザ互換性を確認する
  - Chromeで動作確認をする
  - Safariで動作確認をする
  - Edgeで動作確認をする

### フェーズ5: ドキュメント更新

- [ ] README.mdを更新する
  - デプロイ手順を記載する
  - 環境変数の設定方法を記載する
  - 動作確認手順を記載する
  - トラブルシューティングを記載する

- [ ] ドキュメントを更新する
  - `docs/01_design/07-operations-quality.md`に本番デプロイ手順が追加されていることを確認する
  - 変更履歴を更新する

## 注意事項

- FastAPIサーバーは先にデプロイしてURLを取得する必要がある
- CORS設定は本番環境用に適切に設定する（`allow_origins=["*"]`は非推奨）
- 環境変数はVercelダッシュボードで管理する
- モデルファイルが大きい場合（100MB以上）は外部ストレージを検討する
- デプロイ後は必ず動作確認を実施する
