# 02-05 過去データ取得スクリプト改修（2701-4800回）

## 概要
統合テスト実施のために、`past_results.csv`に全回号分（1-6850回）のデータが必要。現在、2701~4800回のデータが不足している可能性があるため、みずほ銀行のページから取得するスクリプトを改修・実行する。

## 背景
- みずほ銀行のページ形式: `detail.html?fromto=XXXX_YYYY&type=numbers`
- 20回ごとに1ページ（例: 2701_2720, 2721_2740, ...）
- 2701~4800回の範囲を取得する必要がある

## 完了した作業

## 注意事項

- みずほ銀行のページは20回ごとに1ページ
- 2701~4800回の範囲は105ページ分（(4800-2701+1)/20 = 105ページ）
- 取得には時間がかかる可能性がある（1ページあたり1秒待機）
- エラーが5回連続で発生した場合は処理が中断される
- 中間保存が100件ごとに行われるため、途中で中断されてもデータは保存される

## 実行コマンド

```bash
# 2701~4800回のデータを取得
python3 scripts/fetch_past_results.py --from-round 2701 --to-round 4800

# 欠番を確認して取得（必要に応じて）
python3 scripts/fetch_past_results.py --fill-gaps

# NULL値を更新（必要に応じて）
python3 scripts/fetch_past_results.py --update-null
```
