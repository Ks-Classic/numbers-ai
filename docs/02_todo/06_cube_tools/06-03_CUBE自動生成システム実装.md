# CUBE生成システム実装

## 概要

CUBE生成機能をNext.jsアプリケーション内に完全に内包し、ユーザーが回号を入力することで手動トリガーでCUBEを生成する。

## 要件

### CUBE生成の仕様
- **手動トリガー**: ユーザーが回号を入力することで実行される
- **Next.js内包型**: CUBE生成はFastAPIサーバーへの依存なし
- **TypeScript実装**: PythonコードをTypeScriptに移植
- **統一された実装**: 予測アプリとCUBE表示ページの両方で同じ`cube-generator/`を使用（コードの重複を排除）

### データ更新の仕様
- **当選番号データ**: 既存の`scripts/production/auto_update_past_results.py`で自動更新（平日15:00）
- **リハーサル数字**: 当選番号データと同時に取得・更新される
- **CUBE生成は自動化しない**: データ更新のみ自動化、CUBE生成は手動トリガー

## 課題

### 課題1: 平日毎日最新データ（当選番号・リハーサル数字）を自動取得・更新
- **現状**: 既存の`scripts/production/auto_update_past_results.py`で当選番号データは自動更新されている
- **確認事項**: リハーサル数字も同時に取得・更新されているか確認
- **対応**: 必要に応じてリハーサル数字の取得・更新機能を追加

### 課題2: APIデプロイなしでNext.js内にCUBE生成機能を内包
- **現状**: CUBE生成はFastAPIサーバー（`api/main.py`）で実行されている
- **目標**: TypeScriptで再実装し、Next.js API Routeで直接実行
- **メリット**: デプロイ不要、Vercelでそのまま動作

## 解決策

### 課題1の解決策
- 既存の`scripts/production/auto_update_past_results.py`の動作を確認
- リハーサル数字の取得・更新が含まれているか確認
- 必要に応じて`scripts/fetch_past_results.py`を拡張

### 課題2の解決策
- `core/chart_generator.py`のロジックをTypeScriptに移植
- `scripts/generate_extreme_cube.py`のロジックをTypeScriptに移植
- Next.js API Routeで直接実行
- **統一されたディレクトリ**: `src/lib/cube-generator/`に統一し、予測アプリとCUBE表示ページの両方で使用（コードの重複を排除）
