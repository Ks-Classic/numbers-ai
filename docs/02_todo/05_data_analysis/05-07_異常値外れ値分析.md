# 05-07 異常値・外れ値分析

## 概要
統計的に異常なパターンと外れ値の検出。CUBE内に出現しなかった回、リハーサル数字と当選番号が全く重ならなかった回などの異常パターンを検出し、外れ値の特徴を分析する。

## 詳細タスク

### Phase 1: 異常パターンの検出

- [ ] データ読み込みスクリプトを作成する
  - `scripts/analyze_anomalies.py`を作成する
  - `data/past_results.csv`を読み込む
  - 4801-6850回のデータをフィルタリングする
  - 既存の分析結果（CUBE出現分析、リハーサル重なり分析など）を読み込む

- [ ] CUBE内に出現しなかった回を検出する
  - 各回号 × パターン × 対象 × keisen種別について、CUBE内に出現しなかった回を抽出する
  - 出現しなかった回の特徴を分析する（回号、日付、当選番号など）

- [ ] リハーサル数字と当選番号が全く重ならなかった回を検出する
  - 各回号 × 対象(N3/N4)について、リハーサル数字と当選番号が全く重ならなかった回を抽出する
  - 重ならなかった回の特徴を分析する

- [ ] 極端に重なりが多い回を検出する
  - 各回号 × 対象(N3/N4)について、重なり桁数が極端に多い回を抽出する
  - 統計的に外れ値となる回を特定する（Zスコア、IQR法など）

- [ ] 極端に重なりが少ない回を検出する
  - 各回号 × 対象(N3/N4)について、重なり桁数が極端に少ない回を抽出する
  - 統計的に外れ値となる回を特定する（Zスコア、IQR法など）

- [ ] 異常パターンの検出結果を出力する
  - CSV形式で出力する（`data/anomaly_patterns.csv`）
  - JSON形式で出力する（`data/anomaly_patterns.json`）

### Phase 2: 外れ値の分析

- [ ] 統計的に外れ値となる当選番号を検出する
  - 対象(N3/N4)ごとに、統計的に外れ値となる当選番号を検出する
  - Zスコア法、IQR法、マハラノビス距離などを使用する
  - 外れ値の閾値を設定する（例: Zスコア > 3、IQR法の1.5倍など）

- [ ] 外れ値の特徴を分析する
  - 外れ値の数字の組み合わせを分析する
  - 外れ値の出現パターンを分析する
  - 外れ値が出現した回号、日付、曜日などを記録する

- [ ] 外れ値の分類を実施する
  - 外れ値の種類を分類する（極端に大きい値、極端に小さい値など）
  - 外れ値の原因を推測する（データ入力エラー、特殊な抽選など）

- [ ] 外れ値の分析結果を出力する
  - CSV形式で出力する（`data/outliers_analysis.csv`）
  - JSON形式で出力する（`data/outliers_analysis.json`）

### Phase 3: 異常値・外れ値分析レポート作成

- [ ] 分析結果のサマリーを作成する
  - 異常パターンのサマリーを作成する
  - 外れ値のサマリーを作成する
  - 異常値・外れ値の傾向を分析する

- [ ] 可視化を作成する
  - 異常パターンの時系列グラフを作成する
  - 外れ値の分布グラフを作成する
  - 外れ値の特徴の可視化を作成する

- [ ] レポートを出力する
  - Markdown形式でレポートを作成する（`docs/report/anomaly_outlier_analysis_report.md`）
  - 分析結果のサマリー、可視化、統計情報を含める

## 注意事項

- データ範囲: 4801-6850回（約2,050回分）
- 外れ値検出には統計ライブラリが必要（例: `scipy.stats`、`sklearn`）
- 処理時間: 約30分〜1時間
- メモリ使用量: 約1〜2GB
- 出力ファイルは`data/`ディレクトリに保存する
- レポートは`docs/report/`ディレクトリに保存する
