# 05-02 旧keisen vs 新keisen比較分析

## 概要
keisen更新による予測精度への影響評価。旧keisen（1340-6391回）と新keisen（4801-6850回）で生成したCUBEを比較し、当選番号の出現パターンやリハーサル数字との関係性の違いを分析する。

## 詳細タスク

### Phase 1: CUBE生成スクリプトの準備

- [ ] 旧keisenと新keisenのCUBE生成スクリプトを作成する
  - `scripts/generate_cubes_for_comparison.py`を作成する
  - 旧keisenマスターデータ（`data/keisen_master.json`）を読み込む
  - 新keisenマスターデータ（`data/keisen_master_new.json`）を読み込む
  - 各回号について、旧keisenと新keisenでCUBEを生成する
  - **注意**: CUBEはオンデマンド生成（`chart_generator.generate_chart()`を使用）または既存のJSONファイルから読み込む
  - ファイル保存は分析用途のみ（`data/analysis/`ディレクトリ、`.gitignore`に追加）

- [ ] CUBE生成の設定を定義する
  - 対象回号範囲を定義する（4801-6850回）
  - パターン（A1/A2/B1/B2）を定義する
  - 対象（N3/N4）を定義する
  - 各組み合わせでCUBEを生成する

### Phase 2: CUBE内での当選番号出現分析

- [ ] つながり判定ロジックを実装する
  - 縦方向のつながり判定を実装する（連続する行で同じ列）
  - 横方向のつながり判定を実装する（同じ行で連続する列）
  - 斜め方向のつながり判定を実装する（左上→右下、右上→左下）
  - 複合パターン判定を実装する（縦+横、縦+斜めなど）

- [ ] 当選番号がCUBE内に出現したかを判定する
  - 各回号 × パターン × 対象 × keisen種別について、当選番号がCUBE内に出現したかを判定する
  - 出現パターン（縦/横/斜め/複合）を記録する
  - つながりの長さ（連続する数字の数）を記録する
  - つながりの位置（開始行・列、終了行・列）を記録する

- [ ] 出現位置の詳細を記録する
  - 当選番号の各桁がCUBE内のどの位置に出現したかを記録する（座標リスト）
  - 出現した数字の種類（0-9のうち何が出現したか）を記録する
  - 出現回数（同じ数字が複数回出現した場合）を記録する

- [ ] CUBE内出現分析の結果を出力する
  - CSV形式で出力する（`data/cube_appearance_analysis.csv`）
  - JSON形式で出力する（`data/cube_appearance_analysis.json`）
  - 旧keisen vs 新keisenの比較結果を出力する

### Phase 3: リハーサル数字と当選番号の重なり分析

- [ ] 基本重なり統計を集計する
  - 各回号 × 対象(N3/N4)について、リハーサル数字と当選番号の重なりを判定する
  - 重なった桁数（0桁/1桁/2桁/3桁/4桁）を記録する
  - 重なった桁の位置（百の位/十の位/一の位など）を記録する
  - 重なった数字（どの数字が重なったか）を記録する

- [ ] 重なりパターン別集計を実施する
  - 重なり桁数(0-4) × 対象(N3/N4)ごとに出現回数を集計する
  - 出現率（全回数中の割合）を計算する
  - 重なった数字の分布（0-9の出現頻度）を集計する

- [ ] 重なり位置別集計を実施する
  - 重なり位置（各桁） × 対象(N3/N4)ごとに重なり回数を集計する
  - 各桁での重なり率を計算する
  - 重なった数字の分布（各桁ごと）を集計する

- [ ] リハーサル数字重なり分析の結果を出力する
  - CSV形式で出力する（`data/rehearsal_overlap_analysis.csv`）
  - JSON形式で出力する（`data/rehearsal_overlap_analysis.json`）

### Phase 4: N3とN4のリハーサル数字の重なり分析

- [ ] N3とN4のリハーサル数字の重なりを判定する
  - 各回号について、N3とN4のリハーサル数字が重なったかを判定する
  - 重なった桁数（0桁/1桁/2桁/3桁）を記録する
  - 重なった桁の位置（百の位/十の位/一の位など）を記録する
  - 重なった数字（どの数字が重なったか）を記録する

- [ ] 重なりパターン別集計を実施する
  - 重なり桁数(0-3)ごとに出現回数を集計する
  - 出現率（全回数中の割合）を計算する
  - 重なった数字の分布（0-9の出現頻度）を集計する

- [ ] N3/N4リハーサル重なり分析の結果を出力する
  - CSV形式で出力する（`data/n3_n4_rehearsal_overlap_analysis.csv`）
  - JSON形式で出力する（`data/n3_n4_rehearsal_overlap_analysis.json`）

### Phase 5: リハーサル数字重なり時の当選番号傾向分析

- [ ] 重なり時の当選番号分析を実施する
  - リハーサル重なり桁数(0-3) × 対象(N3/N4)ごとに、当選番号にリハーサル重なりの数字が含まれた回数を集計する
  - 当選番号に含まれた重なり数字の数（0-3）を記録する
  - 当選番号に含まれた重なり数字の桁数を記録する
  - 当選番号に含まれた重なり数字の位置を記録する

- [ ] 重なり数字の当選確率を計算する
  - リハーサル重なり桁数 × 対象(N3/N4)ごとに、重なり数字が当選番号に含まれる確率を計算する
  - 重なり桁数別の当選確率を計算する
  - 重なり位置別の当選確率を計算する

- [ ] リハーサル重なり時の当選番号傾向分析の結果を出力する
  - CSV形式で出力する（`data/rehearsal_overlap_winning_tendency.csv`）
  - JSON形式で出力する（`data/rehearsal_overlap_winning_tendency.json`）

### Phase 6: 旧keisen vs 新keisenの比較レポート作成

- [ ] 比較結果のサマリーを作成する
  - CUBE内出現率の比較（旧keisen vs 新keisen）
  - リハーサル数字重なり率の比較
  - N3/N4リハーサル重なり率の比較
  - 重なり数字の当選確率の比較

- [ ] 可視化を作成する
  - CUBE内出現率の比較グラフを作成する
  - リハーサル数字重なり率の比較グラフを作成する
  - 重なり数字の当選確率の比較グラフを作成する

- [ ] 比較レポートを出力する
  - Markdown形式でレポートを作成する（`docs/report/old_vs_new_keisen_comparison_report.md`）
  - 比較結果のサマリー、可視化、統計情報を含める
