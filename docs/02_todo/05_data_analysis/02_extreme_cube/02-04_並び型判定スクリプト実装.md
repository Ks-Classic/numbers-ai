# 02-04 並び型判定スクリプト実装

**Status**: In Progress
**Priority**: High (Extreme Cube)
**Updated**: 2025-12-12

## 概要
極CUBEの1〜5行目における当選数字（N3の3桁）の並び型を自動判定するスクリプトを実装する。
22種類の型（V字型、逆V字型、横一文字型、縦一文字型、L字型4種類、斜め型4種類、コーナー型8種類、ジグザグ型2種類）を判定し、各回号について型を判定して結果を出力する。

**2025-12-12 更新**: 区分分類（上部/中部/下部）を追加し、並び型判定を相対位置で汎用化。

## 前提条件
- [x] 並び型の定義が確定している（`docs/01_design/extreme-cube-pattern-definitions.md`）
- [x] 極CUBE生成スクリプトが実装されている（`scripts/production/generate_extreme_cube.py`）
- [ ] 並び型判定ロジックの実装

### Phase 2: 並び型判定ロジックの実装

**注意**: Phase 1が完了したら、以下の5つのカテゴリは**並行で実装可能**です。各カテゴリは独立しているため、同時に実装を進めることができます。

**実装進捗**:
- ✅ 基本型（4種類）: 完了
- ✅ 斜め型（4種類）: 完了
- ✅ L字型（4種類）: 完了
- ✅ ジグザグ型（2種類）: 完了
- ✅ コーナー型（8種類）: 完了

**実装の優先順位（判定の効率化のため）**:
1. **基本型**（判定が最も簡単）
2. **斜め型**（判定が比較的簡単）
3. **L字型**（判定が中程度）
4. **ジグザグ型**（判定が中程度）
5. **コーナー型**（判定が最も複雑）

**判定順序の最適化**:
- 判定しやすい型から順に判定することで、早期リターンが可能
- 横一文字型、縦一文字型、斜め型は判定が簡単なため、最初に判定する
- V字型、逆V字型、L字型、ジグザグ型、コーナー型は順次判定する

#### 2-1. 基本型（4種類）の判定ロジック

**並行実装可能**: ✅ 他のカテゴリと独立

- [x] 横一文字型（18パターン）- **最優先**（判定が最も簡単）
- [x] 縦一文字型（8パターン）- **最優先**（判定が最も簡単）
- [x] V字型（12パターン）
- [x] 逆V字型（12パターン）

**実装内容**:
- `is_horizontal_straight()`: 横一文字型の判定
  - 同じ行に3桁が連続して並んでいる（列の差が1）
- `is_vertical_straight()`: 縦一文字型の判定
  - 同じ列に3桁が縦に連続して並んでいる（行の差が1）
- `is_v_shape()`: V字型の判定
  - AとCが同じ行で1列を挟んで隣、BがAとCのプラス1行目でAとCの間にある列に存在
- `is_inverted_v_shape()`: 逆V字型の判定
  - AとCが同じ行で1列を挟んで隣、BがAとCのマイナス1行目でAとCの間にある列に存在

**統合**: `classify_pattern()`関数に基本型判定を追加し、最初に判定されるように実装済み

#### 2-2. 斜め型（4種類）の判定ロジック

**並行実装可能**: ✅ 他のカテゴリと独立
**相対位置対応**: ✅ 2025-12-12 汎用化完了

- [x] 斜め右下型（6パターン）
- [x] 斜め左下型（6パターン）
- [x] 斜め右上型（6パターン）
- [x] 斜め左上型（6パターン）

**実装内容**:
- `check_diagonal_patterns()`: 斜め型（4種類）の判定（相対位置で汎用化）
  - 位置をソートし、行・列の変化を確認して斜め型を判定
  - **連続する3行**で行差が1ずつであることを確認（具体的な行番号に依存しない）
  - **斜め右下型**: 列が+1ずつ増える
  - **斜め左下型**: 列が-1ずつ減る
  - **斜め右上型/斜め左上型**: 元の位置順序で方向を判定

**判定ロジック**:
1. 位置をソート（行、列の順）
2. **連続する3行であることを確認**（`row_diff_12 == 1 and row_diff_23 == 1`）
3. 列の差が±1ずつであることを確認
4. 列の変化が一貫しているか確認

**統合**: `classify_pattern()`関数に斜め型判定を追加し、基本型の後に判定されるように実装済み

#### 2-3. L字型（4種類）の判定ロジック

**並行実装可能**: ✅ 他のカテゴリと独立

- [x] └字型（左下L字、14パターン）
- [x] ┘字型（右下L字、14パターン）
- [x] ┌字型（左上L字、14パターン）
- [x] ┐字型（右上L字、14パターン）

**実装内容**:
- `is_l_shape_bottom_left()`: └字型（左下L字）の判定
  - 縦方向が上から下に伸び、横方向が右に伸びるL字
  - 角は縦方向の下の点と横方向の左端が同じ位置
- `is_l_shape_bottom_right()`: ┘字型（右下L字）の判定
  - 縦方向が上から下に伸び、横方向が左に伸びるL字
  - 角は縦方向の下の点と横方向の右端が同じ位置
- `is_l_shape_top_left()`: ┌字型（左上L字）の判定
  - 横方向が右に伸び、縦方向が下に伸びるL字
  - 角は横方向の左端と縦方向の上の点が同じ位置
- `is_l_shape_top_right()`: ┐字型（右上L字）の判定
  - 横方向が左に伸び、縦方向が下に伸びるL字
  - 角は横方向の右端と縦方向の上の点が同じ位置

**判定ロジック**:
1. 3つの位置のうち、2つが同じ行にあり、1つが別の行にあることを確認
2. 横方向の2つが隣接していることを確認（列の差が1）
3. 縦方向が横方向の行の下の行にあることを確認（行の差が1）
4. 角の位置が各L字型の定義に合致していることを確認

**統合**: `classify_pattern()`関数にL字型判定を追加し、斜め型の後に判定されるように実装済み

#### 2-4. ジグザグ型（2種類）の判定ロジック

**並行実装可能**: ✅ 他のカテゴリと独立
**相対位置対応**: ✅ 2025-12-12 汎用化完了

- [x] ジグザグ型（右斜め、7パターン）
- [x] ジグザグ型（左斜め、7パターン）

**実装内容**:
- `is_zigzag_right()`: ジグザグ型（右斜め）の判定（相対位置で汎用化）
  - **連続する3行**であることを確認
  - 最上行と最下行が同じ列にある
  - 中間行が最上行の列+1にある（右斜め下）
- `is_zigzag_left()`: ジグザグ型（左斜め）の判定（相対位置で汎用化）
  - **連続する3行**であることを確認
  - 最上行と最下行が同じ列にある
  - 中間行が最上行の列-1にある（左斜め下）

**判定ロジック**:
1. **連続する3行であることを確認**（具体的な行番号に依存しない）
2. 最上行と最下行が同じ列にあることを確認
3. 右斜め型: 中間行が最上行の列+1
4. 左斜め型: 中間行が最上行の列-1

**統合**: `classify_pattern()`関数にジグザグ型判定を追加し、斜め型の後に判定されるように実装済み
