# 05-01 keisen基礎集計

## 概要
keisen作成のための基礎統計を実施する。4801-6850回のデータから、前々回・前回パターンごとの当選数字の出現頻度とランキングを集計し、新keisenマスターデータを生成する。

## 詳細タスク

### Phase 1: データ準備

- [ ] データ読み込みスクリプトを作成する
  - `scripts/analyze_keisen_base_stats.py`を作成する
  - `data/past_results.csv`を読み込む
  - 4801-6850回のデータをフィルタリングする
  - データクリーニングを実施する（NULL値の除去、形式チェック）

- [ ] 前々回・前回の当選番号を抽出する
  - 各回号について、前回（round_number-1）の当選番号を取得する
  - 各回号について、前々回（round_number-2）の当選番号を取得する
  - N3とN4を別々に処理する
  - 各桁（百の位/十の位/一の位、千の位/百の位/十の位/一の位）を抽出する

### Phase 2: パターン出現頻度分布の集計

- [ ] パターン出現頻度を集計する
  - 各桁 × 前々回(0-9) × 前回(0-9)の組み合わせごとに出現回数をカウントする
  - 全パターン中の出現率を計算する（出現回数 / 全回数 × 100）
  - サンプル数が5回未満のパターンリストを抽出する
  - サンプル数が10回未満のパターンリストを抽出する

- [ ] パターン出現頻度分布の結果を出力する
  - CSV形式で出力する（`data/keisen_pattern_frequency.csv`）
  - JSON形式で出力する（`data/keisen_pattern_frequency.json`）
  - 統計サマリーを出力する（平均出現回数、最小/最大出現回数など）

### Phase 3: 当選数字の分布とランキングの集計

- [ ] 当選数字の出現回数を集計する
  - 各桁 × 前々回(0-9) × 前回(0-9) × 当選数字(0-9)の組み合わせごとに出現回数をカウントする
  - 各パターン内での出現率を計算する（出現回数 / パターン出現回数 × 100）

- [ ] ランキングを作成する
  - 各パターンごとに、当選数字を出現回数の降順でソートする
  - 同率の場合は昇順でソートする
  - 上位3位までの数字リストを抽出する

- [ ] 当選数字の分布とランキングの結果を出力する
  - CSV形式で出力する（`data/keisen_digit_distribution.csv`）
  - JSON形式で出力する（`data/keisen_digit_distribution.json`）
  - ランキング結果を出力する（`data/keisen_digit_ranking.json`）

### Phase 4: 統計的信頼性指標の計算

- [ ] サンプル数を計算する
  - 各パターンごとのサンプル数（N）を計算する
  - サンプル数が不足しているパターンを特定する

- [ ] 標準偏差を計算する
  - 各パターンごとに、当選数字の出現回数の標準偏差を計算する
  - 標本標準偏差を使用する

- [ ] 95%信頼区間を計算する
  - 各パターンごとに、各数字の出現率の95%信頼区間を計算する
  - 正規分布を仮定した信頼区間を使用する

- [ ] 信頼性スコアを計算する
  - サンプル数に基づく信頼性スコアを計算する
  - スコアが低いパターンを特定する

- [ ] 統計的信頼性指標の結果を出力する
  - CSV形式で出力する（`data/keisen_reliability_stats.csv`）
  - JSON形式で出力する（`data/keisen_reliability_stats.json`）

### Phase 5: 新keisenマスターデータの生成

- [ ] keisenマスターデータの構造を定義する
  - 既存の`data/keisen_master.json`の構造を確認する
  - 新keisenマスターデータの構造を定義する（n3/n4、各桁、前々回、前回、予測出目配列）

- [ ] ランキングから予測出目配列を生成する
  - 各パターンごとに、上位3位までの数字リストを予測出目配列として使用する
  - 同率の場合は昇順でソートする
  - サンプル数が不足しているパターンは空配列または警告を付与する

- [ ] 新keisenマスターデータを生成する
  - JSON形式で新keisenマスターデータを生成する（`data/keisen_master_new.json`）
  - 全てのパターン（N3: 300パターン、N4: 400パターン）が含まれていることを確認する
  - JSON形式のバリデーションを実施する

- [ ] 新keisenマスターデータの検証を実施する
  - 全ての必須パターンが定義されていることを確認する
  - 予測出目が0〜9の範囲内であることを確認する
  - 既存のkeisen_master.jsonとの構造の整合性を確認する

### Phase 6: 基礎集計レポートの作成

- [ ] 集計結果のサマリーを作成する
  - パターン出現頻度分布のサマリーを作成する
  - 当選数字の分布とランキングのサマリーを作成する
  - 統計的信頼性指標のサマリーを作成する

- [ ] 可視化を作成する
  - パターン出現頻度のヒストグラムを作成する
  - 当選数字の分布のヒートマップを作成する
  - 信頼性スコアの分布を作成する

- [ ] レポートを出力する
  - Markdown形式でレポートを作成する（`docs/report/keisen_base_stats_report.md`）
  - 集計結果のサマリー、可視化、統計情報を含める
