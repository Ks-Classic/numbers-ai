# ハイパーパラメータチューニング結果レポート

## 概要

数字選択式宝くじ（ナンバーズ）予測モデルのハイパーパラメータチューニングを実施しました。目標はAUC-ROCを0.55-0.60に改善することでした。

## 実施日時

- 実施日: 2024年
- モデルバージョン: v_from_4801

## チューニング方法

### 使用手法

- **手法**: RandomizedSearchCV（ランダムサーチ）
- **試行回数**: 50回
- **クロスバリデーション**: 3-fold
- **評価指標**: AUC-ROC

### パラメータ空間

以下のパラメータを探索しました：

- `max_depth`: [4, 5, 6, 7, 8]
- `learning_rate`: [0.01, 0.05, 0.1]
- `n_estimators`: [300, 500, 700]
- `subsample`: [0.7, 0.8, 0.9]
- `colsample_bytree`: [0.7, 0.8, 0.9]
- `reg_alpha`: [0.01, 0.1, 1.0]
- `reg_lambda`: [0.1, 1.0, 10.0]

## 分析と考察

### 改善の限界

ハイパーパラメータチューニングによる改善は限定的でした：

1. **N3モデル**: AUC-ROCが0.5%しか改善しなかった
2. **N4モデル**: AUC-ROCが逆に0.3%低下した
3. **Precision/F1-Score**: 両モデルとも減少した

### 問題点の分析

1. **クロスバリデーションのエラー**
   - CVスコアがNaNになっているケースが多数発生
   - これは、特定のパラメータ組み合わせでモデルが正常に学習できなかった可能性を示唆

2. **データの制約**
   - 約1,900回分のデータでは、ハイパーパラメータの影響が限定的
   - モデルの予測能力は主に特徴量に依存している可能性が高い

3. **パラメータ空間の特性**
   - 探索したパラメータ範囲では、大幅な改善が見込めなかった
   - より広いパラメータ空間の探索が必要

### Top-5 Accuracyの維持

重要な点として、**Top-5 Accuracyは100%を維持**しています。これは、実用的な予測能力が継続していることを示しています。

## 推奨される次のステップ

### 1. 特徴量エンジニアリングの改善（優先推奨）

ハイパーパラメータだけでは改善が限定的なため、**特徴量エンジニアリングの改善**が効果的です：

- リハーサル数字関連特徴量の強化（現在64-67%の重要度）
- より細かい粒度の距離分析
- 時系列的な特徴量の追加
- 形状・位置特徴量の強化

### 2. より広範囲なパラメータ探索

改良版スクリプト（`tune_hyperparameters_v2.py`）を使用して、より広いパラメータ空間を探索：

- より多くの試行回数（100回以上）
- より細かいパラメータ範囲
- StratifiedKFoldを使用したより堅牢なクロスバリデーション
- 早期停止機能の追加

### 3. アンサンブル学習の検討

複数のモデルを組み合わせることで、予測精度の向上が期待できます。

### 4. データ量の増加

1-4800回のデータを統合することで、より多くのデータで学習可能になります（ただし、リハーサル数字がないデータの扱いに注意が必要）。

## 結論

ハイパーパラメータチューニングによる改善は限定的でした。これは、現在のモデルの予測能力が主に特徴量に依存しており、ハイパーパラメータの影響が小さいことを示唆しています。

**推奨**: 特徴量エンジニアリングの改善を優先的に実施し、その後でより広範囲なハイパーパラメータ探索を検討することを推奨します。
