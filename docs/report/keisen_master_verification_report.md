### 検証方法

1. **データ範囲**: 1340-6391回（週5回制開始以降）
2. **検証ロジック**:
   - 前々回の当選番号の該当桁（例：百の位）
   - 前回の当選番号の該当桁（例：百の位）
   - その組み合わせのときに、実際に当選した数字（その桁）の出現回数を集計

## 判断基準

### 完全一致（MATCH）

`keisen_master.json`の予測出目が、実データの上位（予測出目の数だけ）に**全て含まれている**場合。

**例**:
- keisen_master.jsonの予測出目: `[3, 9, 5]`
- 実データの上位3位: `[3, 9, 5]`
- → **完全一致** ✅

### 部分一致（PARTIAL_MATCH）

`keisen_master.json`の予測出目の**一部**が実データの上位に含まれている場合。

**例**:
- keisen_master.jsonの予測出目: `[5, 4, 3, 8, 9]`
- 実データの上位5位: `[2, 1, 5, 6, 0]`
- 一致している数字: `5`のみ（1つ）
- → **部分一致** ⚠️

### 不一致（MISMATCH）

`keisen_master.json`の予測出目が、実データの上位に**全く含まれていない**場合。

**例**:
- keisen_master.jsonの予測出目: `[3, 9, 5]`
- 実データの上位3位: `[4, 2, 6]`
- 一致している数字: なし
- → **不一致** ❌
