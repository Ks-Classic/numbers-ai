# リハーサル数字と当選番号の関係性分析レポート

## 目的

本レポートは、予測表上におけるリハーサル数字と当選番号の関係性を実データに基づいて分析し、以下の点を明らかにすることを目的としています：

1. **リハーサル数字との距離**が当選確率に与える影響
2. **リハーサル数字との重なり度**が当選確率に与える影響
3. **裏数字の関係性**が当選確率に与える影響
4. 当選した場合とそうでない場合の特徴量の違い

## 概要

- **分析対象**: 4801回分のデータから生成された学習データ（N3: 73,328サンプル、N4: 74,240サンプル）
- **分析期間**: 4801回〜6849回
- **分析対象**: リハーサル数字が存在するサンプルのみを対象
- **分析指標**: 
  - `rehearsal_distance`: リハーサル数字との平均距離（ユークリッド距離）
  - `overlap_count`: リハーサル数字と同じ位置にある回数
  - `inverse_ratio`: リハーサル数字の裏数字が候補数字に含まれる割合

## まとめ

### 主要な発見

1. **リハーサル数字との距離**:
   - N3: 当選時1.10 vs 非当選時1.08（差: 0.02）- 差は非常に小さい
   - N4: 当選時0.88 vs 非当選時0.88（差: 0.00）- 差はほとんどない
   - **結論**: リハーサル数字との距離と当選の関係性は非常に弱い。距離は全体で1.0程度と非常に近い位置にあることが多い。

2. **重なり度**:
   - N3: 当選時1.80 vs 非当選時1.76（差: 0.04）- 差は非常に小さい
   - N4: 当選時2.73 vs 非当選時2.65（差: 0.08）- 差は小さい
   - **結論**: リハーサル数字との重なり度は当選時にやや高い傾向があるが、差は非常に小さい。多くのサンプルで重なりは0（中央値が0）だが、一部では大きな重なりがある（標準偏差が大きい）。

3. **裏数字の割合**:
   - N3/N4共に: 全て0.00
   - **結論**: 裏数字の関係性は**全く存在しない**。全てのサンプルで裏数字の割合は0となっており、この特徴量は予測に寄与していない。

### 予測モデルへの反映

現在の予測モデルでは、これらの関係性特徴量（`rehearsal_distance`, `overlap_count`, `inverse_ratio`）が特徴量として使用されていますが、分析結果から以下のことが判明しました：

1. **`rehearsal_distance`**: 当選時と非当選時で差がほとんどない（N3: 0.02、N4: 0.00）ため、予測への寄与は極めて小さい可能性が高い

2. **`overlap_count`**: 当選時にやや高い傾向があるが、差は非常に小さい（N3: 0.04、N4: 0.08）。多くのサンプルで0であるため、情報量は限定的

3. **`inverse_ratio`**: 全てのサンプルで0となっており、**この特徴量は予測に全く寄与していない**。コード実装に問題がある可能性がある

### 改善提案

1. **`inverse_ratio`の実装確認**: 全てのサンプルで0となっているのは異常。実装ロジックを確認し、正しく計算されているか検証が必要

2. **特徴量重要度の確認**: XGBoostのfeature importanceを確認し、これらの特徴量が実際に予測に寄与しているかを検証

3. **より効果的な関係性特徴量の検討**: 
   - リハーサル数字との相対位置関係（上下左右の関係）
   - リハーサル数字周辺の数字の出現頻度
   - リハーサル数字から見た各数字の方向性（角度）

4. **パターン別分析**: A1/A2/B1/B2パターン別に分析し、パターンによって関係性が異なるかを確認

### 特徴量の定義と計算方法

#### 1. `rehearsal_distance`（リハーサル数字との距離）

**定義**:
- 候補数字（0-9の各数字）の各位置から、最も近いリハーサル数字の位置までのユークリッド距離を計算
- その距離の平均値を特徴量とする

**計算式**:
```
rehearsal_distance = (1/N) × Σ min(√((c_row - r_row)² + (c_col - r_col)²))
```
- N: 候補数字の位置数
- c_pos: 候補数字の位置 (c_row, c_col)
- r_pos: リハーサル数字の位置 (r_row, r_col)

**実際の計算例（6849回N3、数字6の場合）**:
1. 数字6の位置: (1,4), (2,3), (3,5), (4,6), (5,3), (6,4), (7,3), (8,4) = 8箇所
2. リハーサル数字「631」の位置: 22箇所（上記参照）
3. 各6の位置から最も近いリハーサル位置までの距離を計算:
   - (1,4)から最も近いリハーサル位置 = (1,4) itself → 距離 = 0
   - (2,3)から最も近いリハーサル位置 = (2,3) itself → 距離 = 0
   - ...（全ての位置がリハーサル位置と一致）
4. **結果**: `rehearsal_distance = 0.000`（距離0の平均）

**実際の計算例（6849回N3、数字4の場合）**:
1. 数字4の位置: (3,1), (4,2), (5,8), (6,7), (7,2), (7,8), (8,1), (8,7) = 8箇所
2. 各4の位置から最も近いリハーサル位置までの距離を計算:
   - (3,1)から最も近いリハーサル位置 = (3,6) → 距離 = √((3-3)² + (1-6)²) = 5
   - (4,2)から最も近いリハーサル位置 = (4,5) → 距離 = √((4-4)² + (2-5)²) = 3
   - (5,8)から最も近いリハーサル位置 = (5,6) → 距離 = √((5-5)² + (8-6)²) = 2
   - ...（各位置の最小距離を計算）
3. **結果**: `rehearsal_distance = 1.685`（平均距離）

**意味**: 予測表上で候補数字がリハーサル数字にどれだけ近いかを示す。値が小さいほど近い。

#### 2. `overlap_count`（重なり度）

**定義**:
- 候補数字の位置とリハーサル数字の位置が同じ座標にある回数

**計算式**:
```
overlap_count = |{候補数字の位置} ∩ {リハーサル数字の位置}|
```

**実際の計算例（6849回N3、数字6の場合）**:
1. 数字6の位置集合: {(1,4), (2,3), (3,5), (4,6), (5,3), (6,4), (7,3), (8,4)}

3. 共通要素: {(1,4), (2,3), (3,5), (4,6), (5,3), (6,4), (7,3), (8,4)} = 8箇所
4. **結果**: `overlap_count = 8`

**実際の計算例（6849回N3、数字4の場合）**:

3. 共通要素: {}（空集合）
4. **結果**: `overlap_count = 0`

**意味**: 候補数字とリハーサル数字が同じ位置に出現する回数。値が大きいほど重なりが多い。

#### 3. `inverse_ratio`（裏数字の割合）

**定義**:
- リハーサル数字の裏数字（`inverse(n) = (n + 5) % 10`）が候補数字に含まれる割合

**計算式**:
```
inverse_ratio = (裏数字が一致する位置数) / (候補数字の位置数)
```

**実装ロジック**（現在のコード）:
```python
for r_pos in rehearsal_positions:
    rehearsal_digit = grid[r_pos[0]][r_pos[1]]  # リハーサル位置の数字
    inverse_digit = inverse(rehearsal_digit)     # 裏数字を計算
    if (r_pos[0], r_pos[1]) in candidate_positions:  # 同じ位置に候補があるか
        candidate_digit = grid[r_pos[0]][r_pos[1]]   # 候補位置の数字
        if candidate_digit == inverse_digit:         # 裏数字と一致するか
            inverse_count += 1
```

**問題点**:
- 現在の実装では、**リハーサル位置と候補位置が同じ座標にないと計算されない**
- つまり、「リハーサル数字の位置」と「候補数字の位置」が重なっている場合のみチェックする
- しかし、リハーサル数字の位置にはリハーサル数字そのものが存在するため、その位置の裏数字と候補数字が一致することはない（リハーサル数字の裏数字≠リハーサル数字そのもの）
- そのため、**全てのサンプルで0となっている**

**正しい実装（想定）**:
- リハーサル数字の裏数字の位置を取得し、その位置に候補数字があるかを確認すべき

**実際の計算例（6849回N3、リハーサル数字「631」の場合）**:
- リハーサル数字「631」の裏数字: 6→1, 3→8, 1→6 → 「186」
- 裏数字「186」の位置を取得する必要がある
- そして、候補数字の位置と裏数字の位置が一致する割合を計算すべき

**現在の実装の問題**:
- リハーサル位置（6,3,1がある位置）で、その位置の数字が裏数字と一致するかをチェックしている
- しかし、位置(1,4)には数字6があり、その裏数字は1である
- 位置(1,4)に数字1があるかをチェックしているが、実際には位置(1,4)には数字6があるため、一致しない
- つまり、**実装ロジックが間違っている**

**意味**: リハーサル数字の裏数字が当選番号に含まれる傾向を示す（本来）。現在の実装では正しく計算されていない。
