# 02-04 学習データ範囲の選定と再学習

## 概要
学習データ範囲を選定し、AIモデルを再学習する。現在の精度（AUC-ROC: 0.51）を向上させるため、**段階的アプローチ**で改善を実施する。

## 🎯 採用アプローチ: 段階的アプローチ（最新版）

**重要な方針決定**:
1. **特徴量エンジニアリング実験はスキップ**: 全特徴量を使用し、ハイパーパラメータチューニングに集中する
2. **1000回分での分割モデルは検討しない**: データ量が少ないため、統合モデルのみ
3. **4801回分での分割モデルは条件付き**: 統合モデル + ハイパーパラメータチューニング後、精度が目標未達かつパターン間の精度差が大きい場合のみ検討

### Phase 1: ベースライン確立（優先度: 最高）

**目的**: データ範囲を拡大しながらベースラインモデルを作成し、精度の変化を確認する

#### ステップ1: 1000回分でベースラインモデル作成（優先度: 最高）
**目的**: MVP（100回分）から1000回分に拡大し、ベースラインモデルを作成

**実施内容**:
- 既存の特徴量セット（全特徴量）で学習
- 軸数字予測モデル（2モデル）の学習
- 組み合わせ予測モデル（4モデル）の学習
- 精度確認と100回分との比較

**データ範囲**: 5849回 〜 6849回（約1,000件）
**処理時間**: 約2〜3時間

#### ステップ2: 4801回から最新回まででベースラインモデル作成（優先度: 最高）
**目的**: 1000回分から4801回まで拡大し、ベースラインモデルを作成

**実施内容**:
- config.pyを `MIN_ROUND = 4801` に変更
- 同じ特徴量セット（全特徴量）で学習
- 精度確認と1000回分との比較
- データ範囲による精度変化を確認

**データ範囲**: 4801回 〜 6849回（約2,049件）
**処理時間**: 約4〜6時間

### Phase 2: ハイパーパラメータチューニング（Phase 1完了後）

**目的**: 全特徴量を使用してハイパーパラメータを最適化し、精度を向上させる

**方針**: 特徴量エンジニアリング実験はスキップし、全特徴量を使用してハイパーパラメータチューニングに集中する

#### ステップ3: 1000回分でのハイパーパラメータチューニング（統合モデル）
- 全特徴量セットを使用（特徴量エンジニアリング実験はスキップ）
- ハイパーパラメータの探索範囲を定義（03-03）
- グリッドサーチまたはベイズ最適化を実施
- 最適パラメータでモデルを再学習

#### ステップ4: 4801回分でのハイパーパラメータチューニング（統合モデル）
- 全特徴量セットを使用（特徴量エンジニアリング実験はスキップ）
- ハイパーパラメータの探索範囲を定義（03-03）
- グリッドサーチまたはベイズ最適化を実施
- 最適パラメータでモデルを再学習

### Phase 3: 精度評価と判断（Phase 2完了後）

**目的**: 両データ範囲の精度を比較し、次ステップを決定する

#### ステップ5: 精度評価と比較
- 両データ範囲の精度を比較
- AUC-ROC >= 0.65 の達成確認
  - 目標達成 → ✅ 完了（Phase 4へ）
  - 目標未達 → Step 6へ

#### ステップ6: パターン別精度分析（4801回分のみ・条件付き）
**条件**: AUC-ROC < 0.65 の場合のみ実施
- 統合モデルで各パターンの精度を分析
- パターン間の精度差を確認
  - 最大値 - 最小値が0.1以上 → 分割モデルを検討（Step 7へ）
  - 最大値 - 最小値が0.1未満 → 分割モデルは不要

#### ステップ7: 分割モデル（条件付き・4801回分のみ）
**条件**: AUC-ROC < 0.65 かつ パターン間の精度差が0.1以上
- 4801回分で4パターンごとに個別モデルを学習
- 各パターンでハイパーパラメータチューニング
- 統合モデル vs 分割モデルの比較

### Phase 4: 最終モデル生成

**目的**: 最適なパラメータで最終モデルを生成

#### ステップ8: 最終モデル生成
- 最適なモデルとパラメータで再学習
- モデル評価（AUC-ROC目標: 0.65以上）
- モデルを保存・デプロイ
