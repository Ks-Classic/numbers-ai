# 07-02 セキュリティ対策実装

## 概要
セキュリティ対策を実装し、脆弱性を防ぐ。

## 詳細タスク

- [ ] 環境変数の管理を確認する
  - .env.localが.gitignoreに含まれていることを確認する
  - .env.local.exampleを更新する
  - 本番環境の環境変数を設定する

- [ ] APIキーの管理を確認する
  - APIキーがコードにハードコーディングされていないことを確認する
  - 環境変数で管理することを確認する
  - 漏洩リスクを確認する

- [ ] HTTPS設定を確認する
  - VercelでHTTPSが有効になっていることを確認する
  - TLS 1.2以上を使用することを確認する
  - 証明書の有効期限を確認する

- [ ] CORS設定を確認する
  - 許可するオリジンを制限する
  - 不要なメソッドを許可しない
  - セキュアな設定を確認する

- [ ] 入力値のサニタイズを実装する
  - XSS対策を実装する（React自動エスケープ）
  - DOMPurifyを使用する（必要に応じて）
  - SQLインジェクション対策を実装する（Supabase SDK使用）

- [ ] CSRF対策を実装する（Phase 4）
  - トークンベース認証を実装する
  - SameSite Cookie設定を実装する
  - CSRFトークンを実装する

- [ ] レート制限を実装する（Phase 4）
  - Vercel Edge Middlewareを使用する
  - またはupstash/ratelimitを使用する
  - リクエスト制限を設定する

- [ ] セキュリティヘッダーを設定する
  - Content-Security-Policyを設定する
  - X-Frame-Optionsを設定する
  - X-Content-Type-Optionsを設定する
  - Strict-Transport-Securityを設定する

- [ ] 依存関係の脆弱性を確認する
  - npm auditを実行する
  - 脆弱性があれば修正する
  - 定期的に確認する

- [ ] ログの機密情報をマスクする
  - ログにAPIキーが含まれないようにする
  - パスワードが含まれないようにする
  - 個人情報が含まれないようにする

- [ ] セキュリティテストを実施する
  - OWASP Top 10を確認する
  - セキュリティスキャンを実施する
  - 脆弱性があれば修正する
