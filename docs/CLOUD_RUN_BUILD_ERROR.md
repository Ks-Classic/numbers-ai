# Cloud Runビルドエラーの解決方法

## ビルドログの確認方法

エラーメッセージに含まれているビルドログのURLにアクセスして、詳細なエラー内容を確認してください：

```
https://console.cloud.google.com/cloud-build/builds;region=asia-northeast1/f6b3c171-44c4-48c9-938f-9a4be45d72e5?project=168554081899
```

## よくあるビルドエラーの原因と対処法

### 1. COPYコマンドでファイルが見つからない

**症状:** `COPY failed: file not found` エラー

**原因:**
- `.gcloudignore`で必要なファイルが除外されている
- ファイルパスが間違っている

**対処法:**
- `.gcloudignore`を確認し、`api/`、`notebooks/`、`data/`ディレクトリが除外されていないことを確認
- DockerfileのCOPYコマンドのパスを確認

### 2. 依存関係のインストールエラー

**症状:** `pip install`が失敗する

**原因:**
- システムパッケージ（gcc、g++）が不足している
- requirements.txtのパッケージが存在しない

**対処法:**
- Dockerfileに`gcc`と`g++`が含まれていることを確認
- requirements.txtの内容を確認

### 3. メモリ不足エラー

**症状:** `Out of memory` エラー

**原因:**
- ビルド時にメモリが不足している
- データファイルが大きすぎる

**対処法:**
- Cloud Buildのマシンタイプを変更（`--machine-type`オプション）
- データファイルを圧縮するか、外部ストレージから取得する

### 4. ビルドタイムアウト

**症状:** ビルドがタイムアウトする

**原因:**
- ビルド時間が長すぎる（デフォルトは10分）

**対処法:**
- Cloud Buildのタイムアウトを延長
- ビルドを最適化（レイヤーキャッシュの活用）

## 現在の設定確認

以下のファイルが正しく設定されているか確認してください：

1. **Dockerfile** - プロジェクトルートに存在し、正しいパスでファイルをコピーしているか
2. **.gcloudignore** - 必要なファイルが除外されていないか
3. **api/requirements.txt** - 依存関係が正しく記載されているか
4. **data/** - データファイルが存在するか

## ビルドログからエラーを特定する手順

1. 上記のURLにアクセスしてビルドログを確認
2. エラーメッセージを特定（通常は最後の方に表示される）
3. エラーの種類に応じて上記の対処法を適用

## 次のステップ

ビルドログを確認したら、具体的なエラーメッセージに基づいて対処してください。

エラーメッセージが分かれば、より具体的な解決策を提案できます。
