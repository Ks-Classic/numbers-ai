# Numbers-AI 自動データ補完機能

## 概要
6842回の予測時に「前回（回号: 6841）のナンバーズ3当選番号が未登録です」というエラーが発生する問題を解決するため、**自動的に公式サイトから当選番号を取得してデータベースを更新する機能**を実装しました。

## 実装内容

### 1. エラー検知と自動リカバリー
予測API（`api/py/axis.py` と `api/py/combination.py`）に以下の機能を追加しました：

```python
# 予測表生成時にNULL当選番号エラーを検知
try:
    grid, rows, cols = generate_chart(current_df, keisen_master, round_number, best_pattern, target)
except Exception as chart_error:
    if '当選番号が未登録です' in str(chart_error):
        # 自動的にWebから最新データを取得
        fetch_result = fetch_and_update(round_number)
        # データ更新後、予測を再試行
```

### 2. データ取得フロー

1. **エラー検知**: 予測実行中に「当選番号が未登録です」エラーを検知
2. **自動スクレイピング**: `fetch_data.py`の`fetch_and_update()`を呼び出し
3. **公式サイトアクセス**: `https://www.hpfree.com/numbers/rehearsal.html` から最新データを取得
4. **データパース**: BeautifulSoupを使用してHTML解析（既存のロジックを活用）
5. **GitHub更新**: 取得したデータを`data/past_results.csv`に反映、GitHubにコミット
6. **予測再試行**: 更新されたデータで予測を再実行

### 3. フェイルセーフ処理

#### 成功パターン
- データ取得成功 → 予測再試行 → 成功結果を返す

#### 失敗パターン
- **公式サイトにもデータがない場合**:
  ```
  予測エラー: 前回（回号: 6841）のナンバーズ3当選番号が未登録です。
  
  公式サイトにも当選番号が未発表です。当選番号が発表されてからお試しください。
  ```

- **ネットワークエラー等**:
  ```
  予測エラー: 前回（回号: 6841）のナンバーズ3当選番号が未登録です。
  
  自動データ取得にも失敗しました: [エラー詳細]
  ```

### 4. 既存機能との統合

この機能は既存の`/api/predict/route.ts`にあるデータチェック機能（106-126行目）を補完します：

- **既存**: GitHubデータを読み込み時にNULLをチェック、事前にfetch
- **新規**: 予測実行時のエラーを検知し、リアルタイムでfetch

両方のレイヤーで保護されることで、より確実にデータ不足を解消できます。

## 技術的詳細

### データ取得元
- URL: `https://www.hpfree.com/numbers/rehearsal.html`
- スクレイピング: `scripts/production/fetch_latest_simple.py`のロジックを活用
- 正規表現とBeautifulSoupでテーブル解析

### GitHub自動更新
- GitHub API経由でコミット
- コミットメッセージ: `chore: 回号{round_number}予測用データを自動更新 [skip ci]`
- PAT_TOKEN環境変数を使用した認証

### パフォーマンス最適化
- 一度のリクエストで複数パターン（A1, A2, B1, B2）を処理
- データfetchは初回エラー時のみ実行（`data_fetch_attempted`フラグで管理）
- 更新されたCSVをメモリ上で直接使用（ファイルI/O削減）

## 今後の拡張性

この実装により以下が可能になります：

1. **完全自動化**: ユーザーが何もしなくてもシステムが自己修復
2. **プロアクティブ更新**: 定期的なGitHub Actionsと組み合わせて事前にデータを準備
3. **モニタリング**: エラー発生とリカバリー成功をログで追跡可能
4. **スケーラビリティ**: 他の回号でも同様のエラーが発生した場合に対応

## まとめ

この機能により、**NULL当選番号によるエラーが発生した場合でも、ユーザーの介入なしに自動的に公式サイトから最新データを取得し、予測を完了させることができます**。これにより、ユーザー体験が大幅に向上し、手動でのデータメンテナンスが不要になります。
