# 05-03 状態管理とAPI連携

## 概要
Zustandを使用した状態管理を実装し、APIとの連携を実装する。

## 詳細タスク

- [ ] Zustandストアを作成する
  - src/lib/store.tsを作成する
  - 予測状態の型定義を作成する
  - ストアの初期状態を定義する

- [ ] 予測入力状態を管理する
  - roundNumberの状態管理を実装する
  - n3Rehearsalの状態管理を実装する
  - n4Rehearsalの状態管理を実装する
  - 状態更新関数を実装する

- [ ] 予測結果状態を管理する
  - predictionResultの状態管理を実装する
  - 結果取得関数を実装する
  - 結果クリア関数を実装する

- [ ] UI状態を管理する
  - activeTargetの状態管理を実装する（'n3' | 'n4'）
  - activePredictionTypeの状態管理を実装する（'box' | 'straight'）
  - displayModeの状態管理を実装する（'axis' | 'total'）
  - 状態更新関数を実装する

- [ ] API状態を管理する
  - loadingの状態管理を実装する
  - errorの状態管理を実装する
  - 状態更新関数を実装する

- [ ] API呼び出し関数を実装する
  - src/lib/api/predict.tsを作成する
  - 予測実行APIを呼び出す関数を実装する
  - エラーハンドリングを実装する
  - タイムアウト処理を実装する

- [ ] API呼び出しを統合する
  - ZustandストアからAPIを呼び出す処理を実装する
  - ローディング状態を更新する処理を実装する
  - エラー状態を更新する処理を実装する
  - 結果をストアに保存する処理を実装する

- [ ] React Queryを導入する（オプション）
  - @tanstack/react-queryをインストールする
  - QueryClientを設定する
  - useQueryフックを使用する（キャッシュ、リトライ等）

- [ ] エラーハンドリングを実装する
  - APIエラーの処理を実装する
  - ネットワークエラーの処理を実装する
  - タイムアウトエラーの処理を実装する
  - ユーザーフレンドリーなエラーメッセージを表示する

- [ ] パフォーマンス最適化を実施する
  - 不要な再レンダリングを防ぐ
  - メモ化を実装する（useMemo, useCallback）
  - パフォーマンスを測定する
