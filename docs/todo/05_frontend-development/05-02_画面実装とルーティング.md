# 05-02 画面実装とルーティング

## 概要
各画面を実装し、ルーティングを設定する。

## 詳細タスク

- [ ] ホーム画面を実装する
  - src/app/page.tsxを実装する
  - 「新規予測を開始」ボタンを配置する
  - デザインシステムに従ったスタイリングを実装する

- [ ] 回号入力画面を実装する
  - src/app/predict/page.tsxを実装する
  - RoundInputコンポーネントを使用する
  - バリデーションを実装する
  - 「次へ」ボタンを実装する

- [ ] リハーサル数字入力画面を実装する
  - src/app/predict/rehearsal/page.tsxを実装する
  - RehearsalInputコンポーネントを使用する
  - 自動フォーカス移動を実装する
  - 「AI予測を実行」ボタンを実装する

- [ ] ローディング画面を実装する
  - src/app/predict/loading/page.tsxを実装する
  - ローディングコンポーネントを使用する
  - プログレスバーを実装する
  - 処理ステップを表示する

- [ ] 予測結果画面を実装する
  - src/app/predict/result/page.tsxを実装する
  - APIから予測結果を取得する処理を実装する
  - 結果表示コンポーネントを使用する
  - エラー状態の表示を実装する

- [ ] ルーティングを設定する
  - Next.js App Routerを使用する
  - 各画面のルートを設定する
  - リダイレクト処理を実装する（必要に応じて）

- [ ] ナビゲーションを実装する
  - ヘッダーにナビゲーションを追加する
  - アクティブ状態の表示を実装する
  - モバイル対応を実装する

- [ ] 404ページを実装する
  - src/app/not-found.tsxを作成する
  - カスタム404ページを実装する
  - ホームへのリンクを追加する

- [ ] エラーページを実装する
  - src/app/error.tsxを作成する
  - カスタムエラーページを実装する
  - エラー情報を表示する

- [ ] 画面遷移アニメーションを実装する
  - Framer Motionを使用する
  - ページ遷移のアニメーションを実装する
  - スムーズなUXを実現する
