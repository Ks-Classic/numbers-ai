# 03-03 ハイパーパラメータチューニング

## 概要
XGBoostのハイパーパラメータを最適化し、モデルの精度を向上させる。

## 📋 最新の方針（2025-01-XX 決定）

**特徴量エンジニアリング実験はスキップ**: 全特徴量を使用し、ハイパーパラメータチューニングに集中する

**実施対象**:
- **1000回分**: 統合モデル + ハイパーパラメータチューニング（分割モデルは検討しない）
- **4801回分**: 統合モデル + ハイパーパラメータチューニング（分割モデルは条件付きで検討）

## 前提条件

**Phase 1（ベースライン確立）が完了していること**
- 1000回分でベースラインモデル作成完了
- 4801回から最新回まででベースラインモデル作成完了

## 詳細タスク

### ステップ1: 1000回分でのハイパーパラメータチューニング（統合モデル）

**前提条件**: Phase 1のステップ1（1000回分でベースラインモデル作成）が完了していること

- [ ] ハイパーパラメータの探索範囲を定義する
  - max_depth: [4, 5, 6, 7, 8]
  - learning_rate: [0.01, 0.05, 0.1, 0.15]
  - n_estimators: [300, 500, 700, 1000]
  - subsample: [0.7, 0.8, 0.9]
  - colsample_bytree: [0.7, 0.8, 0.9]
  - min_child_weight: [1, 3, 5]
  - gamma: [0, 0.1, 0.2]
  - reg_alpha: [0, 0.1, 0.5]
  - reg_lambda: [1, 1.5, 2]

- [ ] グリッドサーチまたはベイズ最適化を実装する
  - notebooks/05_hyperparameter_tuning.ipynbを作成する
  - グリッドサーチを実装する（小規模な場合）
  - またはOptunaを使用したベイズ最適化を実装する
  - W&B Sweepsを使用する（推奨）

- [ ] 全特徴量セットを使用（特徴量エンジニアリング実験はスキップ）
  - 既存の全特徴量（形状・位置・関係性・集約特徴 + パターンID）を使用

- [ ] 実験を実行する
  - 50〜100パターンの実験を実行する
  - 各実験をW&Bで記録する
  - 実験の進捗を確認する

- [ ] 実験結果を分析する
  - W&Bダッシュボードで結果を確認する
  - 最適なパラメータセットを特定する
  - パラメータの影響を分析する

- [ ] 最適パラメータでモデルを再学習する
  - 最適なパラメータセットを使用する
  - 6つの統合モデルを再学習する
  - 評価を実施する

- [ ] 最適パラメータをドキュメント化する
  - 最適なパラメータセットを記録する
  - README.mdまたはドキュメントに記載する
  - 再現性を確保する

### ステップ2: 4801回分でのハイパーパラメータチューニング（統合モデル）

**前提条件**: Phase 1のステップ2（4801回から最新回まででベースラインモデル作成）が完了していること

- [ ] ハイパーパラメータの探索範囲を定義する
  - ステップ1と同じ探索範囲を使用

- [ ] グリッドサーチまたはベイズ最適化を実装する
  - notebooks/05_hyperparameter_tuning.ipynbを実行（4801回分データで）
  - 全特徴量セットを使用（特徴量エンジニアリング実験はスキップ）

- [ ] 実験を実行する
  - 50〜100パターンの実験を実行する
  - 各実験をW&Bで記録する

- [ ] 実験結果を分析する
  - W&Bダッシュボードで結果を確認する
  - 最適なパラメータセットを特定する

- [ ] 最適パラメータでモデルを再学習する
  - 最適なパラメータセットを使用する
  - 6つの統合モデルを再学習する
  - 評価を実施する

- [ ] 最適パラメータをドキュメント化する
  - 最適なパラメータセットを記録する

### ステップ3: 4801回分での分割モデル検討（条件付き）

**条件**: 
  - AUC-ROC < 0.65 かつ
  - パターン間の精度差が0.1以上

**実施内容**:
- [ ] 4801回分で4パターンごとに個別モデルを学習
  - A1パターン専用モデル
  - A2パターン専用モデル
  - B1パターン専用モデル
  - B2パターン専用モデル

- [ ] 各パターンでハイパーパラメータチューニング
  - 各パターン専用モデルで最適パラメータを探索
  - 上記の探索範囲を使用

- [ ] 統合モデル vs 分割モデルの比較
  - 精度比較
  - 管理コスト比較
  - 最終的なモデル選択

## 注意事項

- **1000回分での分割モデルは検討しない**: データ量が少ないため、統合モデルのみ
- **特徴量エンジニアリング実験はスキップ**: 全特徴量を使用し、ハイパーパラメータチューニングに集中する
- **4801回分での分割モデルは条件付き**: 統合モデル + ハイパーパラメータチューニング後、精度が目標未達かつパターン間の精度差が大きい場合のみ検討
