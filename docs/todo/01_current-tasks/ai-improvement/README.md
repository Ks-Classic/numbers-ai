# AI精度改善タスク

**最終更新日時:** 2025-01-XX XX:XX JST

## 📋 概要

AIモデルの精度を向上させ、目標値（AUC-ROC: 0.65以上）を達成するためのタスクです。

## 🎯 現在の進捗

### Phase 1: ベースライン確立 ✅ 進行中

#### ステップ1: 1000回分でベースラインモデル作成 ✅ 完了

- ✅ データ前処理完了
- ✅ 特徴量エンジニアリング完了
- ✅ モデル学習完了
- ⏳ モデル評価と比較（進行中）

#### ステップ2: 4801回分でベースラインモデル作成 ⏳ 進行中

- ⏳ config.pyを更新（`MIN_ROUND = 4801`）
- ⏳ データ前処理を実行
- ⏳ 特徴量エンジニアリング（全特徴量を使用）
- ⏳ モデル再学習（既存のパラメータを使用）
- ⏳ モデル評価と比較（1000回分との比較）

### Phase 2: ハイパーパラメータチューニング ⏳ 予定

#### ステップ3: 1000回分でのハイパーパラメータチューニング

- ⏳ ハイパーパラメータの探索範囲を定義
- ⏳ グリッドサーチまたはベイズ最適化を実施
- ⏳ 全特徴量セットを使用（特徴量エンジニアリング実験はスキップ）
- ⏳ 最適パラメータでモデルを再学習

#### ステップ4: 4801回分でのハイパーパラメータチューニング

- ⏳ ハイパーパラメータの探索範囲を定義
- ⏳ グリッドサーチまたはベイズ最適化を実施
- ⏳ 全特徴量セットを使用（特徴量エンジニアリング実験はスキップ）
- ⏳ 最適パラメータでモデルを再学習

### Phase 3: 精度評価と判断 ⏳ 予定

#### ステップ5: 精度評価と比較

- ⏳ 両データ範囲の精度を比較
- ⏳ AUC-ROC >= 0.65 の達成確認

#### ステップ6: パターン別精度分析（条件付き）

**条件**: AUC-ROC < 0.65 の場合のみ実施

- ⏳ 統合モデルで各パターンの精度を分析
- ⏳ パターン間の精度差を確認

#### ステップ7: 分割モデル（条件付き・4801回分のみ）

**条件**: AUC-ROC < 0.65 かつ パターン間の精度差が0.1以上

- ⏳ 4801回分で4パターンごとに個別モデルを学習
- ⏳ 各パターンでハイパーパラメータチューニング
- ⏳ 統合モデル vs 分割モデルの比較

## 📝 重要な方針決定

### 特徴量エンジニアリング実験について

**方針**: スキップ（全特徴量を使用）

**理由**:
1. 全特徴量使用の判断: 現在の特徴量セットは既に設計済みで、十分な情報を含んでいる
2. コストパフォーマンス: 特徴量エンジニアリング実験（20-30時間）に対して、ハイパーパラメータチューニングの効果が高い見込み
3. 効率性: まずハイパーパラメータチューニングで精度向上を図り、必要に応じて後から特徴量エンジニアリングを検討

### 分割モデルについて

**方針**:
- 1000回分: 検討しない（データ量が少ないため、統合モデルのみ）
- 4801回分: 条件付き（統合モデル + ハイパーパラメータチューニング後、精度が目標未達かつパターン間の精度差が大きい場合のみ検討）
