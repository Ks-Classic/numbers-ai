# 学習・モデル生成の管理方法について

## 推奨アプローチ: 設定ファイル（config.py）で管理

### 理由

1. **コードの重複がない**
   - 別ファイルに分けると、同じコードが複数ファイルに存在し、メンテナンスが大変
   - 設定ファイルなら、Notebookのコードは1つで済む

2. **柔軟な実験が可能**
   - `config.py`の`TRAIN_SIZE`を変更するだけで、異なる学習範囲での実験が可能
   - 例: 100回分、500回分、1000回分、1800回分など

3. **バージョン管理が簡単**
   - 設定ファイルだけを変更すれば、異なる実験を管理できる
   - Gitで設定変更の履歴を追跡しやすい

4. **再現性が高い**
   - 設定ファイルに実験パラメータが明確に記録される
   - 同じ設定で再実行が容易

### 設定ファイルの使い方

```python
# notebooks/config.py
TRAIN_SIZE = 1000  # 学習範囲を変更するだけで、異なる実験が可能
BASE_ROUND_AUTO = True  # 最新回号を自動取得
```

### 異なる実験を行う場合

1. **設定を変更する方法**（推奨）
   - `config.py`の`TRAIN_SIZE`を変更
   - 同じNotebookで実行
   - 出力ファイル名も自動的に変更される（`train_data_1000.csv`など）

2. **別ファイルに分ける方法**（完全に別のアルゴリズムの場合）
   - アルゴリズム自体が異なる場合のみ推奨
   - 例: 異なる特徴量エンジニアリング手法、異なるモデルアーキテクチャ

## 現在の実装

- ✅ `notebooks/config.py`で学習範囲を管理
- ✅ Notebookファイルは設定ファイルから設定を読み込む
- ✅ 異なる学習範囲での実験が容易に可能

## 学習の実行方法

### 方法1: Jupyter Notebookで実行（推奨）
1000回分のデータ処理には時間がかかるため、Jupyter Notebookで実行することを推奨します。

1. `01_data_preparation.ipynb`を実行
2. `03_feature_engineering.ipynb`を実行（時間がかかります）
3. `04_model_training.ipynb`を実行

### 方法2: Pythonスクリプトで実行
簡易テスト用のスクリプトも用意しました：
- `notebooks/run_01_data_preparation.py`
- `notebooks/run_03_feature_engineering_test.py`（テスト用）
