# 02-02 CSVデータのマイグレーション

## 概要
既存のCSVデータをSupabase PostgreSQLに移行する。

## 詳細タスク

- [ ] CSVデータを読み込むスクリプトを作成する
  - scripts/migrate-csv-to-supabase.tsを作成する
  - CSVパーサーを使用してデータを読み込む処理を実装する
  - データのバリデーションを実装する

- [ ] データ変換処理を実装する
  - CSV行をデータベース行に変換する関数を実装する
  - 日付形式を変換する処理を実装する（YYYY-MM-DD）
  - 数値型を変換する処理を実装する
  - NULL値の処理を実装する

- [ ] バッチインサート処理を実装する
  - 大量データを効率的に挿入する処理を実装する
  - バッチサイズを設定する（100件ずつ等）
  - トランザクション処理を実装する
  - エラーハンドリングを実装する

- [ ] 重複チェック処理を実装する
  - 既存データとの重複をチェックする処理を実装する
  - round_numberで重複を確認する
  - 重複がある場合はスキップまたは更新する処理を実装する

- [ ] マイグレーションスクリプトを実行する
  - 開発環境でテスト実行する
  - データが正しく挿入されることを確認する
  - エラーが発生した場合の処理を確認する
  - 本番環境で実行する

- [ ] データ整合性を確認する
  - CSVの件数とDBの件数が一致することを確認する
  - サンプルデータを抽出して内容を確認する
  - データ型が正しいことを確認する
  - NULL値が正しく処理されていることを確認する

- [ ] インデックスの効果を確認する
  - クエリパフォーマンスを測定する
  - round_numberでの検索速度を確認する
  - draw_dateでの検索速度を確認する
  - インデックスが有効に機能していることを確認する

- [ ] データ読み込み処理を更新する
  - src/lib/data-loader/past-results.tsを更新する
  - CSV読み込み処理をSupabaseクエリに置き換える
  - 型定義を更新する
  - エラーハンドリングを実装する

- [ ] APIの動作確認をする
  - データ読み込みAPIが正常に動作することを確認する
  - レスポンスタイムを測定する（目標1秒以内）
  - エラーハンドリングが正常に動作することを確認する
