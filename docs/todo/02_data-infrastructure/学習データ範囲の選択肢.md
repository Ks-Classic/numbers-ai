# 学習データ範囲の選択肢

## 概要
モデル学習に使用するデータ範囲を選択します。以下の2つの選択肢から選べます。

## 選択肢

### 選択肢1: 1000回分（直近1000回分）

**設定方法:**
```python
# notebooks/config.py
TRAIN_SIZE = 1000  # 1000回分を指定
MIN_ROUND = None   # 未指定
```

**データ範囲:**
- 回号: 5849回 〜 6849回
- データ件数: 約1,000件
- 処理時間: 約2〜3時間（特徴量エンジニアリング含む）

**メリット:**
- ✅ 処理時間が短い
- ✅ 最新データにフォーカス
- ✅ リソース消費が少ない

**デメリット:**
- ⚠️ データ量が少ない（約1,000件）
- ⚠️ 一部のデータにリハーサル数字がない可能性がある

---

### 選択肢2: 4801回から最新回まで（リハーサル数字がある全範囲）

**設定方法:**
```python
# notebooks/config.py
TRAIN_SIZE = None  # Noneに設定
MIN_ROUND = 4801   # 4801回から最新回まで
```

**データ範囲:**
- 回号: 4801回 〜 6849回
- データ件数: 約2,049件（6849-4801+1）
- 処理時間: 約4〜6時間（特徴量エンジニアリング含む）

**メリット:**
- ✅ データ量が約2倍（約2,049件）
- ✅ リハーサル数字があるデータのみを使用可能
- ✅ より多くのデータで学習できるため、精度向上の可能性が高い

**デメリット:**
- ⚠️ 処理時間が長い（約2倍）
- ⚠️ リソース消費が多い

---

## 🎯 推奨アプローチ: 段階的アプローチ

### 第1段階: 1000回分で改善実験（推奨）
**目的**: 特徴量エンジニアリングやハイパーパラメータの調整を素早く検証し、改善手法を確立する

**進め方**:
1. **特徴量エンジニアリング実験**（03-02）
   - 異なる特徴量セットを比較
   - 特徴量重要度を分析
   - 最適な特徴量セットを決定

2. **ハイパーパラメータチューニング**（03-03）
   - XGBoostパラメータの最適化
   - グリッドサーチまたはベイズ最適化を実施
   - 最適なパラメータセットを決定

3. **モデル再学習と評価**
   - 改善された特徴量とパラメータで再学習
   - 改善効果を確認（AUC-ROC目標: 0.65以上）

**メリット**:
- ✅ 処理時間が短い（2〜3時間）
- ✅ 改善手法を素早く検証できる
- ✅ データ量以外の要因（特徴量、パラメータ）を確認できる
- ✅ 問題があればすぐに修正可能

### 第2段階: 4801回から最新回までで最終モデル生成
**目的**: 確立した改善手法を、より多くのデータで適用し、最終モデルを生成する

**進め方**:
1. config.pyを `MIN_ROUND = 4801` に変更
2. データ前処理を実行
3. 第1段階で確立した特徴量セットとパラメータで再学習
4. 最終的なモデル精度を確認
5. モデルを保存・デプロイ

**メリット**:
- ✅ データ量が約2倍（約2,049件）
- ✅ リハーサル数字があるデータのみを使用可能
- ✅ より多くのデータで学習できるため、精度向上の可能性が高い

## 従来の推奨事項（参考）

### 初回学習・テストの場合
- **選択肢1（1000回分）**を推奨
  - 処理が速く、結果を早く確認できる
  - 問題があればすぐに修正可能

### 本番モデル生成の場合
- **選択肢2（4801回から最新回まで）**を推奨
  - より多くのデータで学習できる
  - リハーサル数字があるデータのみを使用可能
  - 精度向上の可能性が高い

**注意**: 現在の精度（AUC-ROC: 0.51）を改善するため、**段階的アプローチ**が推奨されます。

## 設定変更方法

1. `notebooks/config.py`を開く
2. 選択肢に応じて設定を変更:

```python
# 選択肢1の場合
TRAIN_SIZE = 1000
MIN_ROUND = None

# 選択肢2の場合
TRAIN_SIZE = None
MIN_ROUND = 4801
```

3. `notebooks/01_data_preparation.ipynb`を実行
4. データ範囲が正しく設定されていることを確認

## 注意事項

- 設定を変更したら、必ず`01_data_preparation.ipynb`から再実行してください
- 異なるデータ範囲で学習したモデルは別ファイル名で保存されます（`train_data_1000.csv` vs `train_data_from_4801.csv`）
- モデルファイル名も自動的に変更されます（`v1000` vs `v_from_4801`）

## 次のステップ

1. 選択肢を決定
2. `config.py`を更新
3. `01_data_preparation.ipynb`を実行
4. `03_feature_engineering.ipynb`を実行（時間がかかります）
5. `04_model_training.ipynb`を実行
6. モデル評価を実施
