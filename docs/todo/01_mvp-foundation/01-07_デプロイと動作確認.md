# 01-07 デプロイと動作確認

## 概要
Vercelにデプロイし、本番環境で動作確認を行う。

## 詳細タスク

- [ ] Vercelプロジェクト設定を確認する
  - Vercelダッシュボードでプロジェクトを作成する
  - GitHubリポジトリと連携する
  - ビルド設定を確認する（Next.js自動検出）

- [ ] 環境変数を設定する
  - Vercelダッシュボードで環境変数を設定する
  - 必要な環境変数をリストアップする
  - 本番環境とプレビュー環境で分けて設定する
  - 環境変数の値を確認する

- [ ] vercel.jsonを設定する（必要に応じて）
  - カスタムビルド設定を追加する
  - ヘッダー設定を追加する（CORS等）
  - リダイレクト設定を追加する（必要に応じて）

- [ ] ビルドエラーの確認と修正をする
  - ローカルで`pnpm build`を実行する
  - ビルドエラーを修正する
  - TypeScriptエラーを修正する
  - ESLintエラーを修正する

- [ ] 静的ファイルの配置を確認する
  - public/ディレクトリのファイルを確認する
  - favicon.icoが正しく配置されているか確認する
  - 画像ファイルのパスを確認する

- [ ] データファイルの配置を確認する
  - data/ディレクトリのファイルが含まれているか確認する
  - CSVファイルが正しく読み込めるか確認する
  - JSONファイルが正しく読み込めるか確認する
  - モデルファイルが正しく配置されているか確認する（Phase 4まで）

- [ ] 初回デプロイを実行する
  - mainブランチにプッシュする
  - Vercelで自動デプロイが開始されることを確認する
  - ビルドログを確認する
  - デプロイが成功することを確認する

- [ ] 本番環境での動作確認をする
  - デプロイされたURLにアクセスする
  - ホーム画面が正しく表示されることを確認する
  - 回号入力が動作することを確認する
  - リハーサル数字入力が動作することを確認する
  - AI予測が実行されることを確認する
  - 予測結果が正しく表示されることを確認する

- [ ] 全機能の動作確認をする
  - N3/N4切替が動作することを確認する
  - ボックス/ストレート切替が動作することを確認する
  - 軸数字/総合ランキング切替が動作することを確認する
  - 軸数字候補の展開/折りたたみが動作することを確認する
  - 手動指定軸機能が動作することを確認する
  - スコア説明モーダルが動作することを確認する

- [ ] エラーハンドリングの確認をする
  - 不正な入力時のエラー表示を確認する
  - APIエラー時の表示を確認する
  - ネットワークエラー時の表示を確認する
  - エラーメッセージが適切に表示されることを確認する

- [ ] パフォーマンス確認をする
  - 初期ローディング時間を測定する（目標3秒以内）
  - 画面遷移時間を測定する（目標500ms以内）
  - AI予測応答時間を測定する（目標5秒以内）
  - Lighthouseスコアを確認する（目標90+）

- [ ] モバイルデバイスでの動作確認をする
  - スマートフォンでアクセスする
  - 画面サイズが正しく表示されることを確認する
  - タッチ操作が正しく動作することを確認する
  - 入力フィールドが使いやすいことを確認する

- [ ] ブラウザ互換性確認をする
  - Chromeで動作確認をする
  - Safariで動作確認をする
  - Edgeで動作確認をする
  - Firefoxで動作確認をする（必要に応じて）

- [ ] README.mdを更新する
  - デプロイ手順を記載する
  - 環境変数の設定方法を記載する
  - 動作確認手順を記載する
  - トラブルシューティングを記載する

- [ ] ドキュメントを更新する
  - 変更履歴を更新する
  - デプロイ情報を記載する
  - 動作確認結果を記録する
