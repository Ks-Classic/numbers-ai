# 01-01 プロジェクトセットアップと環境構築

## 概要
Next.js 14 + TypeScript + Tailwind CSSのプロジェクト基盤を構築し、開発環境を整備する。

## 詳細タスク

- [ ] Next.js 14プロジェクトを作成する（create-next-app使用）
  - App Routerを使用する
  - TypeScriptを有効化する
  - ESLint設定を追加する

- [ ] Tailwind CSSを導入して設定する
  - tailwind.config.tsを作成する
  - globals.cssにTailwindのベーススタイルを追加する
  - PostCSS設定を確認する

- [ ] shadcn/uiを導入する
  - shadcn/uiの初期化コマンドを実行する
  - components.jsonを設定する
  - 必要なコンポーネントをインストールする（button, input, tabs, card等）

- [ ] プロジェクト構造を作成する
  - src/app/ディレクトリ構造を作成する
  - src/components/ディレクトリ構造を作成する
  - src/lib/ディレクトリ構造を作成する
  - src/types/ディレクトリ構造を作成する

- [ ] 基本的な設定ファイルを整備する
  - tsconfig.jsonをカスタマイズする
  - .eslintrc.jsonを設定する
  - .prettierrcを設定する
  - .gitignoreを確認・更新する

- [ ] パッケージマネージャーをpnpmに設定する
  - pnpmのインストールを確認する
  - package.jsonの依存関係を整理する
  - pnpm-lock.yamlを生成する

- [ ] Vercelプロジェクトの準備をする
  - Vercelアカウントにログインする
  - GitHubリポジトリと連携する
  - 環境変数の設定箇所を確認する

- [ ] README.mdを更新する
  - プロジェクト概要を記載する
  - セットアップ手順を記載する
  - 開発コマンドを記載する

- [ ] 開発環境の動作確認をする
  - `pnpm dev`でローカルサーバーが起動することを確認する
  - ホーム画面が表示されることを確認する
  - ビルドエラーがないことを確認する
