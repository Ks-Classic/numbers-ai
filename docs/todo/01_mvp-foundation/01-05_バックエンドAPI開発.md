# 01-05 バックエンドAPI開発

## 概要
Next.js API RoutesとFastAPIで予測実行APIを実装する。

## 詳細タスク

- [ ] Next.js API Routeの基本構造を作成する
  - src/app/api/predict/route.tsを作成する
  - POSTメソッドのハンドラーを実装する
  - 基本的なレスポンス構造を実装する

- [ ] リクエストバリデーションを実装する
  - Zodスキーマを作成する（PredictRequestSchema）
  - roundNumberのバリデーションを実装する（1-9999の整数）
  - n3Rehearsalのバリデーションを実装する（3桁の数字文字列）
  - n4Rehearsalのバリデーションを実装する（4桁の数字文字列）
  - バリデーションエラーのレスポンスを実装する

- [ ] 予測表生成処理を統合する
  - 回号に基づいて予測表を生成する処理を実装する
  - N3パターンAの予測表を生成する
  - N4パターンAの予測表を生成する
  - 生成エラーのハンドリングを実装する

- [ ] 特徴量計算処理を統合する
  - 予測表とリハーサル数字から特徴量を計算する処理を実装する
  - 各数字（0-9）の特徴量を計算する
  - 各組み合わせの特徴量を計算する
  - 特徴量の正規化・スケーリングを実装する

- [ ] FastAPI AI推論エンジンの準備をする
  - api/ディレクトリを作成する
  - api/main.pyを作成する
  - FastAPIアプリケーションの基本構造を実装する

- [ ] AI推論エンドポイントを実装する
  - /api/predictエンドポイントを実装する
  - リクエストボディの型定義を実装する（Pydantic）
  - モデル読み込み処理を実装する
  - 特徴量を受け取って推論を実行する処理を実装する
  - スコアを算出する処理を実装する（確率 * 1000）

- [ ] 軸数字予測処理を実装する
  - 8つのモデルを読み込む処理を実装する
  - 各数字（0-9）に対する予測を実行する処理を実装する
  - スコア順にランキングする処理を実装する
  - 結果をフォーマットする処理を実装する

- [ ] 組み合わせ予測処理を実装する
  - 指定された軸数字を含む組み合わせを生成する処理を実装する
  - 各組み合わせの特徴量を計算する処理を実装する
  - 組み合わせ予測モデルで推論を実行する処理を実装する
  - スコア順にランキングする処理を実装する

- [ ] APIレスポンス形式を実装する
  - PredictionResult型を定義する
  - N3/N4、ボックス/ストレート、軸数字/組み合わせの構造を実装する
  - generatedAtタイムスタンプを追加する処理を実装する
  - 成功レスポンスのフォーマットを実装する

- [ ] エラーハンドリングを実装する
  - バリデーションエラーの処理を実装する（400 Bad Request）
  - データが見つからない場合のエラー処理を実装する（404 Not Found）
  - モデルエラーの処理を実装する（500 Internal Server Error）
  - その他の内部エラーの処理を実装する（500 Internal Server Error）
  - エラーレスポンスのフォーマットを統一する

- [ ] API統合テストを実装する
  - 正常系のテストケースを書く
  - 異常系のテストケースを書く（不正な入力、データなし等）
  - APIテストを実行して確認する
  - Postmanまたはcurlで手動テストをする

- [ ] パフォーマンス最適化を実施する
  - レスポンスタイムを測定する（目標5秒以内）
  - モデル読み込みを最適化する（キャッシュ等）
  - 不要な処理を削減する
  - 並列処理が可能な箇所を特定する
