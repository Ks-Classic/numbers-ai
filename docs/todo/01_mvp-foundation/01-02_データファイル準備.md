# 01-02 データファイル準備

## 概要
過去実績データ（CSV）と罫線マスターデータ（JSON）を準備し、データ読み込み処理を実装する。

## 詳細タスク

- [ ] past_results.csvファイルを作成する
  - CSVのカラム定義を確認する（round_number, draw_date, n3_winning, n4_winning, n3_rehearsal, n4_rehearsal）
  - 直近100回分のデータを収集する
  - CSVフォーマットを検証する（数値、日付形式等）

- [ ] keisen_master.jsonファイルを作成する
  - 08-keisen-master-creation.mdを参照して作成手順を理解する
  - 紙媒体の罫線ルールを入手する
  - N3用の罫線マスターデータを構造化する
    - 百の位の予測出目ルールを定義する（前回・前々回の組み合わせ×10×10パターン）
    - 十の位の予測出目ルールを定義する（前回・前々回の組み合わせ×10×10パターン）
    - 一の位の予測出目ルールを定義する（前回・前々回の組み合わせ×10×10パターン）
  - N4用の罫線マスターデータを構造化する
    - 千の位の予測出目ルールを定義する（前回・前々回の組み合わせ×10×10パターン）
    - 百の位の予測出目ルールを定義する（前回・前々回の組み合わせ×10×10パターン）
    - 十の位の予測出目ルールを定義する（前回・前々回の組み合わせ×10×10パターン）
    - 一の位の予測出目ルールを定義する（前回・前々回の組み合わせ×10×10パターン）
  - JSONスキーマを定義する（各桁の予測出目ルール）
  - JSON形式のバリデーションを実施する

- [ ] keisen_master.jsonの検証を実施する
  - JSON構文エラーがないことを確認する
  - 全ての必須パターンが定義されていることを確認する（N3: 300パターン、N4: 400パターン）
  - 予測出目が0〜9の範囲内であることを確認する
  - 既知の回号で予測表を生成して動作確認する
  - 過去の実績と照合する（可能な場合）

- [ ] データファイルを配置する
  - data/ディレクトリを作成する
  - past_results.csvをdata/に配置する
  - keisen_master.jsonをdata/に配置する

- [ ] CSV読み込みユーティリティを作成する
  - src/lib/data-loader/past-results.tsを作成する
  - CSVパーサーを実装する（またはcsv-parseライブラリを使用）
  - 型定義を追加する（RoundNumber, PastResult等）

- [ ] JSON読み込みユーティリティを作成する
  - src/lib/data-loader/keisen-master.tsを作成する
  - JSONファイルを読み込む関数を実装する
  - 型定義を追加する（KeisenMaster, ColumnRules等）

- [ ] データバリデーションを実装する
  - 回号の範囲チェック（1-9999）を実装する
  - 日付形式のチェックを実装する
  - 当選番号のフォーマットチェック（3桁/4桁）を実装する
  - リハーサル数字のNULL許容チェックを実装する

- [ ] エラーハンドリングを実装する
  - ファイルが見つからない場合のエラー処理を実装する
  - データフォーマットエラーの処理を実装する
  - エラーメッセージを日本語で表示する

- [ ] データ読み込みのテストを実装する
  - 正常系のテストケースを書く
  - 異常系のテストケースを書く（ファイルなし、不正フォーマット等）
  - テストを実行して確認する
