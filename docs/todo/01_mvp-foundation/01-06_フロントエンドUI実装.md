# 01-06 フロントエンドUI実装

## 概要
ユーザーインターフェースを実装し、APIと連携して予測結果を表示する。

## 詳細タスク

- [ ] ホーム画面を実装する
  - src/app/page.tsxを実装する
  - 「新規予測を開始」ボタンを配置する
  - デザインシステムに従ったスタイリングを実装する
  - ナビゲーションリンクを追加する

- [ ] 回号入力画面を実装する
  - src/app/predict/page.tsxを実装する
  - 4桁の数字入力フィールドを実装する
  - 数字キーボードを自動表示する処理を実装する
  - 入力バリデーションを実装する（4桁必須）
  - 「次へ」ボタンを実装する（4桁入力時のみ有効化）
  - 入力中の桁数を表示する処理を実装する
  - エラーメッセージ表示を実装する

- [ ] リハーサル数字入力画面を実装する
  - src/app/predict/rehearsal/page.tsxを実装する
  - N4リハーサル入力フィールドを実装する（4桁、各桁独立）
  - N3リハーサル入力フィールドを実装する（3桁、各桁独立）
  - 桁位置のラベルを表示する（千、百、十、一）
  - 自動フォーカス移動を実装する（数字入力時に次へ）
  - Backspaceで前のフィールドへ戻る処理を実装する
  - 入力済み桁の視覚的区別を実装する
  - 「AI予測を実行」ボタンを実装する（全桁入力時のみ有効化）
  - エラーメッセージ表示を実装する

- [ ] ローディング画面を実装する
  - src/app/predict/loading/page.tsxを実装する
  - プログレスバーを実装する
  - AI分析中のメッセージを表示する
  - 処理ステップの表示を実装する（予測表生成、特徴量計算等）
  - スピナーアニメーションを実装する

- [ ] 予測結果表示画面を実装する
  - src/app/predict/result/page.tsxを実装する
  - APIから予測結果を取得する処理を実装する
  - エラー状態の表示を実装する
  - ローディング状態の表示を実装する

- [ ] N3/N4切替タブを実装する
  - shadcn/uiのTabsコンポーネントを使用する
  - N3タブとN4タブを実装する
  - アクティブタブに応じて結果を切り替える処理を実装する
  - 状態管理を実装する（ZustandまたはuseState）

- [ ] ボックス/ストレート切替タブを実装する
  - サブタブとして実装する
  - ボックスタブとストレートタブを実装する
  - アクティブタブに応じて結果を切り替える処理を実装する

- [ ] 軸数字/総合ランキング切替を実装する
  - 表示モード切り替えボタンを実装する
  - 軸数字モードの表示を実装する
  - 総合ランキングモードの表示を実装する
  - モード切り替え時の状態管理を実装する

- [ ] 軸数字候補表示を実装する
  - 軸数字候補リストを表示するコンポーネントを実装する
  - スコア順にランキング表示する処理を実装する
  - Accordionコンポーネントで展開/折りたたみを実装する
  - 各軸数字の詳細情報を表示する処理を実装する

- [ ] 当選番号候補テーブルを実装する
  - 軸数字候補を展開した際の候補番号テーブルを実装する
  - ランク、番号、スコアを表示するテーブルを実装する
  - スコア順にソートする処理を実装する
  - テーブルのスタイリングを実装する

- [ ] 総合ランキング表示を実装する
  - 全候補をスコア順に表示するテーブルを実装する
  - ページネーションを実装する（必要に応じて）
  - フィルタリング機能を実装する（オプション）

- [ ] 手動指定軸機能を実装する
  - 手動指定軸入力フィールドを実装する（0-9の数字）
  - 「計算」ボタンを実装する
  - 指定軸を含む候補を計算して表示する処理を実装する
  - 結果をテーブルで表示する処理を実装する

- [ ] スコア説明モーダルを実装する
  - shadcn/uiのDialogコンポーネントを使用する
  - スコアの意味を説明するモーダルを実装する
  - 「スコア説明」ボタンを実装する
  - モーダルの開閉処理を実装する

- [ ] 状態管理を実装する
  - Zustandストアを作成する（src/lib/store.ts）
  - 回号、リハーサル数字、予測結果を管理する
  - UI状態（アクティブタブ等）を管理する
  - API状態（loading, error）を管理する

- [ ] エラーハンドリングを実装する
  - APIエラーの表示を実装する
  - ネットワークエラーの処理を実装する
  - タイムアウトエラーの処理を実装する
  - ユーザーフレンドリーなエラーメッセージを表示する

- [ ] レスポンシブデザインを実装する
  - モバイルファーストでデザインする
  - 最小画面サイズ320px×568pxに対応する
  - タブレット・デスクトップのレイアウトを実装する
  - Tailwindのレスポンシブユーティリティを使用する

- [ ] アクセシビリティ対応を実装する
  - セマンティックHTMLを使用する
  - ARIAラベルを適切に設定する
  - キーボード操作に対応する
  - スクリーンリーダー対応を確認する

- [ ] アニメーションを実装する
  - Framer Motionを導入する
  - 画面遷移のアニメーションを実装する
  - ローディングアニメーションを実装する
  - スムーズなUXを実現する
