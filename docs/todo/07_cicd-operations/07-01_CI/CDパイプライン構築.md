# 07-01 CI/CDパイプライン構築

## 概要
GitHub Actionsを使用したCI/CDパイプラインを構築する。

## 詳細タスク

- [ ] GitHub Actionsワークフローを作成する
  - .github/workflows/ci.ymlを作成する
  - トリガー条件を設定する（push, pull_request）
  - 実行環境を設定する（ubuntu-latest）

- [ ] Lintジョブを実装する
  - ESLintを実行する処理を実装する
  - Prettierを実行する処理を実装する
  - エラーがあれば失敗させる

- [ ] TypeScript型チェックジョブを実装する
  - TypeScriptの型チェックを実行する処理を実装する
  - エラーがあれば失敗させる

- [ ] テストジョブを実装する
  - ユニットテストを実行する処理を実装する
  - 統合テストを実行する処理を実装する
  - テストカバレッジを記録する

- [ ] ビルドジョブを実装する
  - Next.jsのビルドを実行する処理を実装する
  - ビルドエラーがあれば失敗させる
  - ビルド成果物を保存する（オプション）

- [ ] Vercel自動デプロイを設定する
  - VercelとGitHubを連携する
  - プレビューデプロイを設定する（PR時）
  - 本番デプロイを設定する（main push時）

- [ ] Cloud Run自動デプロイを設定する（Phase 4）
  - GitHub ActionsでCloud Runにデプロイする処理を実装する
  - Dockerイメージをビルドする処理を実装する
  - Container Registryにプッシュする処理を実装する
  - Cloud Runにデプロイする処理を実装する

- [ ] デプロイ通知を実装する（オプション）
  - Slack通知を実装する
  - メール通知を実装する
  - デプロイ結果を通知する

- [ ] ワークフローをテストする
  - ローカルでワークフローをテストする（act等）
  - GitHubで実際に実行して確認する
  - エラーがあれば修正する
