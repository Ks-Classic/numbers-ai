## 1. ドキュメント目的と適用範囲

### 1.1 目的
本ドキュメントは、ナンバーズ（Numbers 3/4）の当選番号を独自の「中国罫線」理論とAI技術を用いて予測するWebアプリケーションシステムの要件を定義する。本書により、開発チーム全員が共通認識を持ち、実装・テスト・検証を効率的に進めることを目的とする。

### 1.2 適用範囲
- 対象システム: ナンバーズAI予測Webアプリケーション（以下「本システム」）
- 対象抽選: ナンバーズ3、ナンバーズ4
- 対象ユーザー: モバイルデバイスからアクセスする一般エンドユーザー
- 開発期間: MVP（Minimum Viable Product）として7日間で初期バージョンを完成

### 1.3 除外事項
本バージョンでは以下を対象外とする：
- ユーザー認証・会員管理機能
- 課金・決済機能
- データ分析ダッシュボード
- 管理者機能

## 2. ゴール定義

### 2.1 誰が
- **プライマリ**: 個人開発者（1名）
- **セカンダリ**: 将来的には複数のエンドユーザー

### 2.2 何を
独自の「中国罫線」理論に基づく予測表を自動生成し、AIモデルが過去約6,700回分のデータから学習したパターンを活用して、ナンバーズ3/4の当選番号候補をスコア順にランキング表示するWebアプリケーションを構築する。

### 2.3 どれくらいの時間で
- **MVP版**: 7日間で基本機能を実装し、Vercelにデプロイ
- **完全版**: MVP検証後、3ヶ月以内に全機能を段階的に実装

### 2.4 達成基準
- ユーザーが回号とリハーサル数字を入力すると、5秒以内にAI予測結果が表示される
- N3/N4、ボックス/ストレート、0なし/0ありの全パターンで予測が実行される
- モバイルデバイスで快適に操作できるUI/UXが実現される

## 5. プロジェクト概要

### 5.1 ビジョン
独自の「中国罫線」分析理論を、データサイエンスとAI技術によって客観的かつ再現可能な「予測科学」へと昇華させる、世界に一つだけのパーソナル意思決定支援システムを構築する。

### 5.2 プロジェクト背景
- 紙媒体で管理されていた「中国罫線」理論をデジタル化
- 過去約6,700回分の抽選データを活用
- 第4954回以降（約1,800回分）に存在するリハーサル数字を分析材料として追加
- AIによる多角的なパターン学習により予測精度を向上

### 5.3 対象システム
- **システム名**: ナンバーズAI予測システム（Numbers-AI）
- **システム種別**: Webアプリケーション（PWA対応）
- **対象デバイス**: スマートフォン（iOS/Android）、タブレット、PC
- **対象ブラウザ**: Chrome, Safari, Edge（最新版）

### 5.4 開発アプローチ
MVP（Minimum Viable Product）から段階的に機能を拡張するアジャイル開発を採用：

**MVP（Phase 1）**: 7日間
- N3/N4の基本予測機能
- 0なし（パターンA）のみ
- 直近100回分のデータで学習
- CSVベースのデータ管理

**Phase 2**: MVP+2週間
- 学習データを1,800回分に拡大
- 予測精度の向上

**Phase 3**: MVP+1ヶ月
- 全6,700回分のデータで学習
- アンサンブル学習の実装

**Phase 4**: MVP+3ヶ月
- 0あり（パターンB）の実装
- 統一モデルの構築
- Supabase/GCP Cloud Runへの移行

## 6. ペルソナ定義

### 6.1 プライマリペルソナ: 予測分析ユーザー

**基本情報**
- 名前: 田中 健太（仮名）
- 年齢: 35-50歳
- 職業: 会社員
- ITリテラシー: 中〜高（スマートフォンアプリの日常的利用）

**利用動機**
- ナンバーズの当選確率を高めたい
- データに基づく客観的な予測を求めている
- 直感ではなく論理的な根拠が欲しい

**利用シーン**
- 抽選日の午前中（リハーサル数字公開後）
- 通勤時間や休憩時間にスマートフォンから利用
- 1回の利用時間: 3〜5分程度

**ニーズ**
- 素早く予測結果を確認したい
- 複雑な操作は避けたい
- 予測の根拠が知りたい
- 過去の履歴は不要（当日の予測のみ重要）

**ペインポイント**
- 複雑な理論を理解するのは難しい
- 時間がないのに操作が煩雑だと使わなくなる
- ログインなどの面倒な手続きは避けたい

### 7.2 詳細ユーザーアクション

**Step 1: 回号入力**
1. ユーザーがホーム画面の「新規予測を開始」をタップ
2. 回号入力画面が表示される
3. 数字キーボードで4桁の回号を入力
4. 「次へ」ボタンをタップ

**Step 2: リハーサル数字入力**
1. N4の入力フィールドに4桁の数字を入力
2. N3の入力フィールドに3桁の数字を入力
3. 各桁は自動的に次のフィールドにフォーカス移動
4. 「AI予測を実行」ボタンをタップ

**Step 3: 結果表示**
1. AIが分析を実行（3〜5秒）
2. 予測結果画面が表示される
3. デフォルトは「N3 / ボックス / 軸数字」モード
4. ユーザーはタブを切り替えて各種予測を確認
5. 軸数字候補をタップして展開・詳細を確認
6. 手動指定軸で独自の分析も可能

### 8.2 機能詳細仕様

#### F-01: 回号入力機能

**目的**: ユーザーが予測対象の抽選回を指定する

**入力**
- 回号（4桁の数字、例: 6701）

**処理**
- 数値のみ入力可能
- 4桁に満たない場合は「次へ」ボタンを無効化
- 入力中はリアルタイムで桁数を表示

**出力**
- 入力された回号をセッションに保存
- リハーサル数字入力画面へ遷移

**バリデーション**
- 必須入力
- 4桁の数字（0000〜9999）
- 数字以外の入力は不可

**エラーハンドリング**
- 無効な入力時はエラーメッセージを表示
- 「回号は4桁の数字で入力してください」

#### F-02: リハーサル数字入力機能

**目的**: AI予測の重要な手がかりとなるリハーサル数字を入力する

**入力**
- N4リハーサル数字（4桁、千百十一の位）
- N3リハーサル数字（3桁、百十一の位）

**処理**
- 各桁は独立した入力フィールド
- 数字入力時に自動的に次のフィールドへフォーカス移動
- Backspaceキーで前のフィールドへ戻る
- モバイルでは数字キーボードを自動表示

**出力**
- 入力されたリハーサル数字をセッションに保存
- AI予測実行をトリガー

**バリデーション**
- N4: 4桁すべて必須
- N3: 3桁すべて必須
- 各桁は0〜9の数字

**UI/UX要件**
- 視覚的に桁位置が明確（千、百、十、一のラベル表示）
- 入力済みの桁は視覚的に区別
- フォーカス中のフィールドはハイライト

#### F-03: 予測表自動生成機能

**目的**: 中国罫線理論に基づく予測表を自動生成する

**入力**
- 回号
- パターンタイプ（A: 0なし / B: 0あり）
- 対象（N3 / N4）

**処理フロー**
1. 前回・前々回の当選番号を取得
2. 罫線マスターデータから予測出目を抽出
3. 元数字リストを作成（パターン別ルール適用）
4. メイン行を組み立て（4要素×N行）
5. グリッド初期配置（8列、偶数行・偶数列）
6. パターンBの場合は中心に0を配置
7. 裏数字ルール適用（縦パス→横パス、収束まで）
8. 余りマスルール適用（収束まで）

**出力**
- 2次元配列の予測表データ（8列×N行）
- 各セルは0〜9の数字

**詳細アルゴリズム**
docs/表作成ルール.,md の仕様に完全準拠

**パフォーマンス要件**
- 1表の生成時間: 100ms以内

---

#### F-04: AI軸数字予測機能

**目的**: AIが学習したパターンから、当選確率が高い軸数字を提案する

**入力**
- 予測表データ（パターンA/B）
- リハーサル数字
- 対象（N3 / N4）
- 予測タイプ（ボックス / ストレート）

**処理**
1. 予測表から特徴量を計算
   - 形状特徴（線の長さ、曲がり、直線度）
   - 位置特徴（重心、端との距離、集中度）
   - 関係性特徴（リハーサルとの距離、重なり、角度）
   - 集約特徴（密集度、パターン頻度）
2. 学習済みAIモデルに特徴量を入力
3. 各軸数字（0〜9）の当選確率スコアを算出
4. スコア順にソート

**出力**
- 軸数字候補リスト（降順）
  ```typescript
  {
    digit: number;        // 軸数字（0〜9）
    score: number;        // スコア（0〜1000）
    confidence: number;   // 信頼度（0〜100%）
    source: 'A' | 'B';   // 出所（0なし/0あり）
  }[]
  ```

**スコアリング基準**
- スコア範囲: 0〜1000
- 高スコア: 当選確率が高い
- スコア差が50以上: 有意な差
- スコア差が10以下: ほぼ同等

---

#### F-05: AI組み合わせ予測機能

**目的**: 選択された軸数字を含む当選番号候補を提案する

**入力**
- 予測表データ
- リハーサル数字
- 軸数字（ユーザー選択 or AI提案）
- 対象（N3 / N4）
- 予測タイプ（ボックス / ストレート）

**処理**
1. 軸数字を含む全組み合わせを生成
   - N3: 軸1つを含む3桁の組み合わせ
   - N4: 軸1つを含む4桁の組み合わせ
2. 各組み合わせの特徴量を計算
3. 学習済みAIモデルで各組み合わせをスコアリング
4. スコア順にソート
5. 上位候補を返却

**出力**
- 当選番号候補リスト（降順）
  ```typescript
  {
    numbers: string;      // 番号（例: "147"）
    score: number;        // スコア（0〜1000）
    confidence: number;   // 信頼度（0〜100%）
    source: 'A' | 'B';   // 出所
    rank: number;         // 順位
  }[]
  ```

**表示上限**
- 軸数字モード: 各軸の上位20件
- 総合ランキングモード: 上位100件

#### F-06〜F-12: UI機能群

**F-06: N3/N4切替機能**
- タブUIで即座に切替
- 選択中のタブは視覚的に強調
- 切替時はデータを再取得せず、既存データを表示

**F-07: ボックス/ストレート切替機能**
- サブタブUIで切替
- 予測ロジックが異なるため、切替時にデータを再計算

**F-08: 軸数字/総合ランキング切替機能**
- ボタンUIで表示モードを切替
- 軸数字モード: 軸ごとのアコーディオン表示
- 総合モード: 全候補のフラットな一覧表示

**F-09: 軸数字候補展開機能**
- アコーディオンUIで展開/折りたたみ
- 展開時に該当軸を含む候補を表形式で表示
- 同時に複数の軸を展開可能

**F-10: 手動指定軸機能**
- ユーザーが任意の数字（0〜9）を入力
- 「計算」ボタンで該当軸の候補を算出
- 結果は既存の軸候補と同じ形式で表示

**F-11: ローディング表示機能**
- AI分析中はプログレスバーとメッセージを表示
- 処理ステップごとにメッセージを更新
- 推定残り時間を表示

**F-12: スコア説明機能**
- スコアラベルをタップするとモーダル表示
- スコアの意味、算出方法、見方を平易に説明
- モーダル外タップまたは閉じるボタンで閉じる

## 12. 品質基準

### 12.1 コード品質基準

**禁止事項**
- TODOコメントをcommit（issueで管理）
- 未確定ワード（「多分」「おそらく」等）の使用
- ハードコーディング（マジックナンバー、URL等）
- console.logの本番環境への混入

**推奨事項**
- 能動態で記述（「〜される」ではなく「〜する」）
- 明確な命名（略語を避ける）
- 単一責任の原則（1関数1機能）
- DRY原則（Don't Repeat Yourself）

### 12.2 テスト品質基準

**テストケース作成基準**
- 正常系テスト: 全機能必須
- 異常系テスト: 入力バリデーション必須
- 境界値テスト: 数値入力箇所必須
- パフォーマンステスト: 主要機能必須

**テスト実行基準**
- ユニットテスト: commit前に実行
- 統合テスト: PR作成時に実行
- E2Eテスト: デプロイ前に実行
- 全テストパス: マージ条件

### 12.3 ドキュメント品質基準

**記述ルール**

- 英語: 固有名詞、技術用語のみ

- 図表番号: 推奨
- 更新履歴: 必須
