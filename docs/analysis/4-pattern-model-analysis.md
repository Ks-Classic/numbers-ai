# 4パターン統合モデル vs 分割モデル 分析レポート

## 結論：統合モデルを推奨

## 1. 4パターンの定義

### パターン分類
| パターンID | 欠番補足 | 中心0配置 | 説明 |
|-----------|---------|----------|------|
| A1 | する（0〜9全追加） | しない | 現在のパターンA |
| A2 | する（0〜9全追加） | する | 新規 |
| B1 | しない（0のみ追加） | しない | 現在のパターンB（中心0配置あり版） |
| B2 | しない（0のみ追加） | する | 現在のパターンB |

## 2. 統合モデル vs 分割モデルの比較

### 統合モデル（推奨）

**利点：**
1. **データ量が4倍になる**
   - 100回分のデータ → 400回分相当の学習データ
   - より多くのパターンを学習できる
   - 特にデータが少ない場合（MVPの100回分）に有効

2. **パターン間の共通パターンを学習**
   - 4パターンに共通する予測パターンを発見できる
   - 「どのパターンでも通用する法則」を学習

3. **パターン情報を特徴量として追加**
   - パターンID（A1=0, A2=1, B1=2, B2=3）を特徴量に追加
   - パターン特有の特徴も学習可能
   - XGBoostがパターン間の違いも自動的に学習

4. **汎化性能が高い**
   - より少ないデータでも汎化性能が高いモデルになる可能性
   - 未知のパターンにも対応しやすい

5. **同じ評価基準で比較可能**
   - 4パターンの予測結果を同じスコアで比較できる
   - 「どのパターンが良かったか」が明確

**注意点：**
- パターン間の差異が大きい場合、分割の方が精度が高い可能性
- ただし、まずは統合モデルで試し、精度が低い場合のみ分割を検討

### 分割モデル（4つに分ける）

**利点：**
1. パターンごとに最適化できる
2. パターン間の差異が大きい場合、より高い精度が期待できる
3. どのパターンが良かったかが明確

**欠点：**
1. **データ量が1/4になる**
   - 100回分のデータ → 各パターン25回分相当
   - データが少なくなり、過学習のリスクが高い
   - MVPの100回分では特に不利

2. **共通パターンを学習できない**
   - 各パターンで独立に学習するため、共通パターンを活用できない

3. **管理が複雑**
   - 4つのモデルファイルを管理する必要がある
   - デプロイや更新が複雑になる

## 3. 推奨アプローチ

### Phase 1: 統合モデルで試行（MVP）

**実装：**
- パターンIDを特徴量として追加（A1=0, A2=1, B1=2, B2=3）
- 1回の予測で4パターンすべてを生成・評価
- 最も高いスコアのパターンを返す

**特徴量の例：**
```python
features = [
    ...予測表の特徴量（100次元）...,
    pattern_id,  # 0, 1, 2, 3のいずれか
    ...その他の特徴量...
]
```

**メリット：**
- データ量が増える（100回 → 400回相当）
- パターン間の共通パターンを学習できる
- どのパターンが良かったかも分かる（スコアで比較）

### Phase 2: 評価と改善

**評価指標：**
- AUC-ROCで精度を確認
- 各パターンごとの精度も確認

**精度が低い場合：**
- データ量を増やす（1,800回分など）
- 特徴量エンジニアリングを改善
- ハイパーパラメータチューニング

**それでも精度が低い場合：**
- 分割モデルを検討（ただし、データ量が多い場合のみ）

## 4. 実装方針

### 統合モデルの実装

```python
# 特徴量生成時のパターンID追加
def create_features(chart_data, pattern_id):
    features = []
    
    # 予測表の特徴量（100次元）
    features.extend(extract_shape_features(chart_data))
    features.extend(extract_position_features(chart_data))
    features.extend(extract_relation_features(chart_data))
    features.extend(extract_aggregate_features(chart_data))
    
    # パターンID（4次元のone-hot encoding）
    pattern_onehot = [0, 0, 0, 0]
    pattern_onehot[pattern_id] = 1
    features.extend(pattern_onehot)
    
    return features
```

### 推論時の処理

```python
# 4パターンすべてを生成して評価
patterns = [
    {'id': 0, 'name': 'A1', 'fill_gaps': True, 'center_zero': False},
    {'id': 1, 'name': 'A2', 'fill_gaps': True, 'center_zero': True},
    {'id': 2, 'name': 'B1', 'fill_gaps': False, 'center_zero': False},
    {'id': 3, 'name': 'B2', 'fill_gaps': False, 'center_zero': True},
]

results = []
for pattern in patterns:
    chart = generate_chart(round_number, pattern)
    features = create_features(chart, pattern['id'])
    score = model.predict_proba(features)[1]
    results.append({
        'pattern': pattern['name'],
        'score': score,
        'chart': chart
    })

# 最も高いスコアのパターンを返す
best_result = max(results, key=lambda x: x['score'])
```

## 6. 最終的な推奨

✅ **統合モデルを採用する**

**理由：**
1. データ量が増える（特にMVPの100回分では重要）
2. パターン間の共通パターンを学習できる
3. どのパターンが良かったかも分かる（ユーザーの質問に対応可能）
4. まずは統合モデルで試し、精度が低い場合のみ分割を検討

**実装の優先順位：**
1. まず統合モデルで開発・評価
2. 精度が低い場合はデータ量を増やす（1,800回分など）
3. それでも精度が低い場合のみ分割モデルを検討

**結論：確率が最優先という要件を満たすため、統合モデルが最適です。**
