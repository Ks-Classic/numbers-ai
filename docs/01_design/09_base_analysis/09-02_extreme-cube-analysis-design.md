## 1. 概要と適用範囲

### 1.1 極CUBEの定義
- **生成ルール**: リハーサル数字とは独立した独自ルール
- **特徴**:
  - **N3のみ**（N4は対象外）
  - **1パターンのみ**
  - **全期間一貫性**: 初回から最新回まで同じルールで生成可能
  - **5行目の余りマスを0で埋める**（固有ルール）

### 1.2 分析の目的
- リハーサル数字に依存しない純粋な数字パターンの発見
- 曜日や月などの時系列要素と「並び型」の相関検証
- 新しい予測ロジックの確立

### 1.3 スクリプト配置
- **ディレクトリ**: `scripts/base_statistics/02_extreme_cube/`
- **出力先**: `data/analysis_results/02_extreme_cube/`

### 1.4 データ範囲
- **全期間基礎集計**: 1-6850回（全期間、リハーサル数字非依存）
- **並び型分析**: 150回、1000回、2000回、全期間での検証

## 2. 全期間基礎集計

**概要**: リハーサル数字非依存の全期間（1-6850回）統計分析。
**ステータス**: ❌ 未実装

### 2.1 極CUBE内での当選番号出現分析
- **つながり判定**: 縦・横・斜め

### 2.2 数字の出現パターン（全期間）

## 3. 極CUBE並び型分析

**概要**: 極CUBEの2,3,4行目における当選数字（N3の3桁）の並び型を分類し、時系列要素での出現傾向を分析する。
**ステータス**: ❌ 未実装（設計済み）

### 3.1 背景と仮説
- 過去150回の観察で、曜日ごとに特定の「型」（V字型など）が出やすい傾向が見られた。
- サンプル数を1000回、2000回と増やしてもこの傾向が維持されるかを検証する。

### 3.2 並び型の分類
1. **基本型 (Phase 1)**
   - V字型、逆V字型
   - L字型、逆L字型
   - 横一文字型、縦一文字型
   - 斜め型（右下/左下）
   - コーナー型
2. **細分化 (Phase 2)**
   - 有意な傾向が見られた型のみ、行・列の位置でさらに細分化

### 3.3 分析フロー
1. **定義**: 各型の幾何学的定義を確定（例：V字型＝A,Cが同行1列空き、Bがその間の下行）
2. **判定**: スクリプトによる自動判定
3. **集計**: 曜日・月・週ごとの出現分布を集計
4. **検定**: カイ二乗検定等で統計的有意性を確認
5. **ルール化**: 有意なパターンを予測ルールとして採用

## 4. 区切り別比較分析

**概要**: システム変更や環境変化のタイミングでの傾向変化を分析。
**ステータス**: ❌ 未実装

### 4.1 比較対象の区切り
1. **週5実施開始前後** (1340回前後, 2004年)
2. **リハーサル数字導入前後** (4801回前後, 2024年)
3. **最新5年間** (5000回以降)

### 4.2 分析項目
- 当選番号の数字分布の変化
- CUBE内出現率の変化
- 周期性の変化
