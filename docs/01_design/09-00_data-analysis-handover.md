## 1. 背景と経緯

### 1.1 プロジェクトの現状

本プロジェクトは、ナンバーズ（N3/N4）の当選番号を予測するAIシステムの開発プロジェクトです。

**開発の経緯**:
- **初期段階**: 予測モデル（ステップ4: モデリング）を先に実装
- **現在**: 基礎集計（ステップ2: EDA/基礎集計）を改めて実施する段階
- **課題**: 基礎集計が未完了のままモデリングに進んでしまったため、データ理解が不十分

### 1.2 極CUBE並び型分析の背景

**発見**:
- 過去150回程度の観察で、曜日ごとに各型（V字型、逆V字型など）の当選確率に傾向が見られる
- 極CUBEの2,3,4行目における当選数字の並び型に、時系列要素（曜日、月、週など）での傾向がある可能性

**目的**:
- サンプル数を拡張（150回→1000回→2000回）した際の傾向の持続性を検証
- 曜日以外の要素（月、週、回数の区切りなど）での傾向も検証
- 統計的に有意な傾向を特定し、実用的な予測ルールを導出

### 1.3 ドキュメント構造の現状

**現在の構造**:
- `docs/01_design/09-data-analysis-design.md`: 通常CUBEと極CUBEの両方が混在
- `docs/02_todo/05_data_analysis/`: 分析タスクが一覧形式で管理

**問題点**:
- 通常CUBEと極CUBEの前提条件が異なるのに、同じファイルに混在
- 予測モデル構築の5ステップが明示されていない
- 共通分析と個別分析の区別が不明確

## 2. 現状の問題点と課題

### 2.1 予測モデル構築プロセスの問題

**標準的な予測モデル構築プロセス**:
1. **ステップ1: データ収集と準備** - データを収集し、結合する
2. **ステップ2: EDA/基礎集計** - データの異常を発見し、変数間のパターンを理解する
3. **ステップ3: 前処理・特徴量エンジニアリング** - データをクリーンアップし、特徴量を作成する
4. **ステップ4: モデリング** - 予測モデルを構築する
5. **ステップ5: 評価と展開** - モデルの性能を評価し、実運用環境に展開する

**本プロジェクトの現状**:
- ✅ **ステップ1**: 一部完了（`02_data_infrastructure/`）
- ⚠️ **ステップ2**: **未完了**（keisen基礎集計のみ実装済み）
- ✅ **ステップ3**: 完了（特徴量エンジニアリング実装済み、72次元）
- ✅ **ステップ4**: 一部完了（モデル学習済み、AUC-ROC 0.51程度）
- ⚠️ **ステップ5**: 不明確（評価は実施済み、展開はVercelに完了）

**問題点**:
- **ステップ2（EDA/基礎集計）が未完了のまま、ステップ3-4に進んでいる**
- データ理解が不十分な状態でモデルを構築したため、精度向上の余地がある
- 基礎集計の結果を活用した特徴量改善ができていない

### 2.2 通常CUBEと極CUBEの混在問題

**通常CUBEの特徴**:
- リハーサル数字あり（4801回以降）
- 4パターン（A1/A2/B1/B2）対応
- N3/N4両対応
- keisenマスターに依存

**極CUBEの特徴**:
- リハーサル数字なし（全期間で一貫性）
- 1パターンのみ（B1相当）
- N3のみ
- keisenマスターに依存（ただし生成ルールが異なる）

**問題点**:
- 前提条件が異なるのに、同じドキュメントに混在している
- 分析項目が重複している部分（共通分析）と、個別に実施すべき部分の区別が不明確
- 実装時の混乱を招く可能性がある

### 2.3 ドキュメント構造の問題

**現状**:
- `09-data-analysis-design.md`に通常CUBEと極CUBEが混在
- 5ステップの全体フローが明示されていない
- ToDo構造が分析項目ごとに分割されているが、通常CUBE/極CUBE/共通の区別がない

**問題点**:
- 後任が全体像を把握しにくい
- 実装優先順位が不明確
- 共通分析と個別分析の実装が重複する可能性

## 4. 通常CUBEと極CUBEの整理状況

### 4.1 通常CUBEの特徴と分析項目

**通常CUBEの定義**:
- keisen_master.jsonから予測出目を抽出
- 前回・前々回の当選番号を基に生成
- リハーサル数字を特徴量として使用可能
- 4パターン（A1/A2/B1/B2）に対応
- N3/N4両対応

**通常CUBE専用の分析項目**:
1. **keisen基礎集計**（✅ 実装済み）
   - パターン出現頻度分布
   - 当選数字の分布とランキング
   - 統計的信頼性指標

2. **旧keisen vs 新keisen比較分析**（❌ 未実装）
   - 旧keisen（1340-6391回）と新keisen（4801-6850回）の比較
   - CUBE内での当選番号出現分析
   - リハーサル数字と当選番号の重なり分析

3. **CUBE特性分析**（❌ 未実装）
   - CUBEの構造特性（サイズ、密度、メイン行の特性）
   - CUBEと当選番号の関係

4. **相関分析**（❌ 未実装）
   - リハーサル数字と当選番号の相関
   - N3とN4の相関
   - 前回・前々回との相関

### 4.2 極CUBEの特徴と分析項目

**極CUBEの定義**:
- リハーサル数字とは独立した生成ルール
- 全期間（初回1回目～）で一貫性がある
- リハーサル数字が存在しない期間も分析可能
- N3のみ、1パターンのみ
- 5行目の余りマスを0で埋める（極CUBE固有）

**極CUBE専用の分析項目**:
1. **全期間基礎集計**（❌ 未実装）
   - 極CUBE内での当選番号出現分析
   - 数字の出現パターン
   - 時系列・周期性分析

2. **並び型分析（2,3,4行目）**（❌ 未実装、設計済み）
   - 並び型の定義と分類（V字型、逆V字型、L字型など）
   - 並び型判定スクリプトの作成
   - 曜日ごとの型の出現傾向分析
   - 曜日以外の要素での傾向分析
   - 階層的分析の実施

3. **区切り別比較分析**（❌ 未実装）
   - 週5実施開始前後比較（1340回前後）
   - リハーサル数字導入前後比較（4801回前後）
   - 最新5年間分析（5000回以降）

### 4.3 共通分析項目

**通常CUBEと極CUBEの両方で実施すべき分析**:
1. **数字の出現パターン**（❌ 未実装）
   - 数字(0-9)の出現頻度
   - 各桁位置での数字分布
   - よく出現する数字の組み合わせ

2. **時系列・周期性分析**（❌ 未実装）
   - 曜日分析
   - 月相（月齢）分析
   - 季節・月分析
   - 年・四半期分析

3. **数値パターン分析**（❌ 未実装）
   - 連続性・周期性分析
   - 数字の組み合わせ特性

4. **異常値・外れ値分析**（❌ 未実装）
   - 異常パターンの検出
   - 外れ値の分析

### 6.3 分析内容の分類

#### 共通で実施すべき分析

**理由**: 通常CUBEと極CUBEの両方で実施可能で、前提条件に依存しない

**分析項目**:
1. **数字の出現パターン**
   - 数字(0-9)の出現頻度（全体、各桁位置ごと）
   - よく出現する数字の組み合わせ
   - 数字の分布（正規分布か、偏りがあるか）

2. **時系列・周期性分析**
   - 曜日分析（月/火/水/木/金）
   - 月分析（1-12月）
   - 月の第何週目（1-5週目）
   - 年・四半期分析
   - 季節分析（春/夏/秋/冬）

3. **数値パターン分析**
   - 連続性分析（同じ数字が連続して出現）
   - 周期性分析（特定の数字が周期的に出現）
   - 数字の組み合わせ特性

4. **異常値・外れ値分析**
   - 統計的に異常なパターンの検出
   - 外れ値の特徴分析

#### 通常CUBE専用の分析

**理由**: リハーサル数字あり、4パターン対応、N3/N4両対応という前提条件に依存

**分析項目**:
1. **keisen基礎集計**（✅ 実装済み）
   - パターン出現頻度分布
   - 当選数字の分布とランキング
   - 統計的信頼性指標

2. **旧keisen vs 新keisen比較分析**
   - keisen更新による影響評価
   - CUBE内での当選番号出現分析
   - リハーサル数字と当選番号の重なり分析

3. **CUBE特性分析**
   - CUBEの構造特性（サイズ、密度、メイン行の特性）
   - CUBEと当選番号の関係

4. **相関分析**
   - リハーサル数字と当選番号の相関
   - N3とN4の相関
   - 前回・前々回との相関

#### 極CUBE専用の分析

**理由**: リハーサル数字なし、1パターンのみ、N3のみ、全期間で一貫性があるという前提条件に依存

**分析項目**:
1. **全期間基礎集計**
   - 極CUBE内での当選番号出現分析（1-6850回）
   - 数字の出現パターン（全期間）
   - 時系列・周期性分析（全期間）

2. **並び型分析（2,3,4行目）**（設計済み）
   - 並び型の定義と分類（V字型、逆V字型、L字型など）
   - 並び型判定スクリプトの作成
   - 曜日ごとの型の出現傾向分析（150回、1000回、2000回、全期間）
   - 曜日以外の要素での傾向分析（月、週、回数の区切り、年・四半期）
   - 階層的分析の実施（基本型→細分化）

3. **区切り別比較分析**
   - 週5実施開始前後比較（1340回前後）
   - リハーサル数字導入前後比較（4801回前後）
   - 最新5年間分析（5000回以降）

## 8. 重要な注意事項

### 8.1 データ範囲の違い

**通常CUBE**:
- keisen基礎集計: 4801-6850回（リハーサル数字が有効な範囲）
- 旧keisen比較: 1340-6391回（旧keisen） vs 4801-6850回（新keisen）

**極CUBE**:
- 全期間基礎集計: 1-6850回（全期間、リハーサル数字非依存）
- 並び型分析: 150回、1000回、2000回、全期間での検証

### 8.2 リハーサル数字の扱い

**通常CUBE**:
- リハーサル数字あり（4801回以降）
- リハーサル数字を特徴量として使用
- リハーサル数字と当選番号の重なり分析が可能

**極CUBE**:
- リハーサル数字なし（全期間で一貫性）
- リハーサル数字に依存しない分析が可能
- リハーサル数字が存在しない期間（1-4800回）も分析可能

### 8.3 並び型分析の前提条件

**重要**:
- 当選数字の3桁は必ず横・斜め・縦のいずれかでつながっている
- 2行目と4行目だけに出現（3行目を飛ばす）のはNG
- 3列と5列目だけに出現（4列目を飛ばす）のはNG

**V字型の定義（確定）**:
- AとCが同じ行で1列を挟んで隣（例：2行目2列と2行目4列）
- BがAとCのプラス1行目でAとCの間にある列に存在する
- 全パターン: 12パターン（AとCが2行目に6パターン、AとCが3行目に6パターン）

**他の型の定義**:
- V字型と同様に、最短で質問して定義を確定する必要がある
- 階層的分析アプローチを採用（最初は粗い分類、有意な型のみ細分化）
