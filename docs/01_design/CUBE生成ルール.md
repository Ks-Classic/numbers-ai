## 1. 概要

本仕様は、ナンバーズ予測の「CUBE」生成アルゴリズムを定義する。CUBEには**通常CUBE**と**極CUBE**の2種類がある。

### 通常CUBE
- **対象**: N3, N4
- **パターン**: A1, A2, B1, B2（4パターン）
- **用途**: 本番予測、パターン選択、詳細分析

### 極CUBE
- **対象**: N3のみ
- **パターン**: B1相当（1パターンのみ）
- **用途**: 全期間分析、統計分析、リハーサル数字非依存の一貫性のある分析

**主な違い**: 極CUBEは「5行目の余りマスを0で埋める」点のみが異なる。

---

## 2. 入力と前提

- 入力: `source_list`（整数の配列, 0〜9。昇順ソート済み、重複あり）
- 罫線/リハーサルの扱い:
  - 罫線（過去当選番号由来）から「予測出目」を得る
  - リハーサル数字はCUBE生成に影響を与えない（AI特徴量でのみ使用）

### 通常CUBEと極CUBEの違い

| 項目 | 通常CUBE | 極CUBE |
|------|---------|--------|
| **対象** | N3, N4 | N3のみ |
| **パターン** | A1, A2, B1, B2（4パターン） | B1相当（1パターンのみ） |
| **欠番補足** | パターン依存（A1/A2: あり、B1/B2: なし） | なし（B1/B2と同じ） |
| **中心0配置** | パターン依存（A2/B2: あり、A1/B1: なし） | なし（B1と同じ） |
| **最大行数** | 8行（メイン行4本まで） | 5行（メイン行3本まで） |
| **5行目の余りマス** | 余りマスルールで補完（真上のマスをコピー） | **0で埋める**（極CUBE固有） |

---

## 3. パターン別の元数字リスト作成

### 通常CUBE（4パターン）

| パターンID | 欠番補足 | 中心0配置 | 説明 |
|-----------|---------|----------|------|
| **A1** | する（0〜9全追加） | しない | 欠番をすべて補足し、中心0配置なし |
| **A2** | する（0〜9全追加） | する | 欠番をすべて補足し、中心0配置あり |
| **B1** | しない | しない | 欠番を補足せず、中心0配置なし |
| **B2** | しない | する | 欠番を補足せず、中心0配置あり |

- **パターンA1/A2（欠番補足あり）**: 抽出した「予測出目」に、0〜9 の欠番をすべて追加する
- **パターンB1/B2（欠番補足なし）**: 抽出した「予測出目」をそのまま使用（0も含めて、すべて欠番補足しない）
- その後、配列を昇順にソートする（重複は許容）

### 極CUBE（B1相当）

- **欠番補足なし**: 0も含めて、すべて欠番補足しない（予測出目をそのまま使用）
- **中心0配置なし**: B1と同じ

## 4. メイン行の組み立て

通常CUBEと極CUBEで同じロジック。

- 基本思想:
  1) メイン行は1〜4要素を持つ（4要素未満の場合も許容）
  2) リストに残った数字は、一つも無駄にしない
  3) 可能な限り「4種類の数字で構成する」という原則を守る
  4) 4要素未満のメイン行の余りマスは、後続の「メイン行配置後の余りマスルール」で補完される

- アルゴリズム:
  1) `temp_list ← source_list`（コピー）
  2) `main_rows ← []`
  3) while `temp_list` が空でない:
     - `unique_digits ← temp_list` のユニークを昇順で取得
     - if `|unique_digits| ≥ 4`:
       - `members ← unique_digits[0..3]`（構成メンバー）
       - `new_row ← temp_list` を先頭から走査し、`members` の各値を1個ずつ取り出して順に格納
       - `main_rows.push(new_row)`（4要素）
     - else (`|unique_digits| < 4`):
       - `new_row ← unique_digits` の全てをベースとする
       - **重要**: 最大値を繰り返し追加しない。`unique_digits` の要素数分だけを `new_row` とする
       - `new_row` で使った数字を、`temp_list` から削除する
       - `main_rows.push(new_row)`（1〜3要素）

- 結果: `main_rows` は1本以上、各行は1〜4要素。全数字を消費。

**極CUBEの特別処理**: メイン行が3本を超える場合は、4本目以降の数字を3本目に統合（最大3本まで）。

---

## 5. グリッド形状と初期配置

- 列数: 常に 8 列固定（1-indexedでは 1..8列）
- 行数:
  - **通常CUBE**: 最大 8 行（メイン行4本まで: 1, 2, 3, 4, 5, 6, 7, 8行目）
  - **極CUBE**: 最大 5 行（メイン行3本まで: 1, 2, 3, 4, 5行目）

- 配置マスの規則:
  - **行・列ともに1から始まる**（0行目・0列目は使用しない）
  - 初期配置は「奇数行（1, 3, 5, 7行目）」にメイン行を配置
  - 列は「奇数列（1, 3, 5, 7列目）」に配置
  - 1行目に `main_rows[0]`、3行目に `main_rows[1]`、5行目に `main_rows[2]`、7行目に `main_rows[3]`（あれば）を配置

- メイン行配置後の余りマスルール（裏数字適用前）:
  - 目的: メイン行が4要素未満の場合、空の奇数列・偶数列を補完する
  - 対象: 奇数行（1, 3, 5, 7行目）でメイン行が配置されている場合のみ
  - ルール:
    - 奇数行の奇数列（1, 3, 5, 7列目）が空の場合、その上のメイン行（1つ前の奇数行）の同じ列の数字をコピー
    - 奇数行の偶数列（2, 4, 6, 8列目）が空の場合、その上のメイン行（1つ前の奇数行）の同じ列の数字をコピー
  - 実行タイミング: メイン行配置後、中心0配置（A2/B2）の前

- **極CUBEの特別処理**: メイン行配置後の余りマスを0で埋める
  - 対象: 奇数行（メイン行）の奇数列（1, 3, 5, 7列目）で空いているマス
  - 処理: 空のマスに0を配置
  - 実行タイミング: メイン行配置後、裏数字適用前

- パターンA2/B2の特別処理（中心0配置）:
  - **通常CUBEのみ**: パターンA2/B2の場合、中心マス群に0を1つ配置
  - **極CUBE**: 中心0配置なし（B1と同じ）
  - 実行タイミング: メイン行配置後の余りマスルールの後

## 6. 裏数字ルールと充填フェーズ

通常CUBEと極CUBEで同じロジック。

- 裏数字の定義: `inverse(n) = (n + 5) % 10`（例: 0→5, 5→0）
- フェーズ順序: 「縦パス 収束まで」→「横パス 収束まで」→終了
- セル更新の原則: 既に値があるセルは上書きしない（重複禁止）

### 6.1 縦パス（収束まで）

- スキャン順: 行 1..H-1、列 0..W-1（上→下・左→右）
- ルール: `grid[y][x]` が空で、かつ `y>0` かつ `grid[y-1][x]` に値がある場合、`grid[y][x] ← inverse(grid[y-1][x])`
- これを1パスとし、1パスで1つでも更新が起きたら再度縦パスを繰返す。更新がゼロになれば収束。

### 6.2 横パス（収束まで）

- スキャン順: 行 1..H-1、列 0..W-1（上→下・左→右）
- ルール: `grid[y][x]` が空で、かつ `x>0` かつ `grid[y][x-1]` に値がある場合、`grid[y][x] ← inverse(grid[y][x-1])`
- これを1パスとし、1パスで1つでも更新が起きたら再度横パスを繰返す。更新がゼロになれば収束。

---

## 7. 出力

- 出力: `grid`（2次元配列, サイズ = `rows × 8`）
- 先頭行はメイン行0を反映、以下同様に奇数行にメイン行、偶数行は最終的に充填される

## 8. 実行順序のまとめ

### 通常CUBE生成の実行順序

1. パターン別の元数字リスト作成（§3）
2. メイン行の組み立て（§4）
3. グリッド初期配置（§5）
4. メイン行配置後の余りマスルール（§5）
5. 中心0配置（A2/B2のみ、§5）
6. 裏数字ルール（縦パス・収束まで、§6.1）
7. 裏数字ルール（横パス・収束まで、§6.2）

### 極CUBE生成の実行順序

1. パターン別の元数字リスト作成（B1相当、§3）
2. メイン行の組み立て（最大3本まで、§4）
3. グリッド初期配置（最大5行まで、§5）
4. メイン行配置後の余りマスを0で埋める（極CUBE固有、§5）
5. 裏数字ルール（縦パス・収束まで、§6.1）
6. 裏数字ルール（横パス・収束まで、§6.2）

## 10. 使用シーン

### 通常CUBEを使用する場合

- **本番予測**: リハーサル数字を考慮した予測が必要な場合
- **パターン選択**: A1/A2/B1/B2の4パターンから選択したい場合
- **N4対応**: N4の予測も必要な場合
- **詳細分析**: パターンごとの違いを分析したい場合

### 極CUBEを使用する場合

- **全期間分析**: リハーサル数字非依存で全期間（1-6850回）の分析を行いたい場合
- **統計分析**: サンプル数を最大化して統計的信頼性を高めたい場合
- **一貫性**: 全期間で一貫性のあるCUBE生成が必要な場合

---

## 11. 実装指針

- データ型: `number | null` の2次元配列を使用し、空白は `null` とする
- 反復の停止条件: 各パスで更新カウンタを持ち、0 になれば収束
- 行・列は1から始まる（1-indexedで統一）
- テスト: 再現性（同じ入力→同じ出力）、収束性（有限回で停止）、全セル非null（メイン行1本以上前提）
