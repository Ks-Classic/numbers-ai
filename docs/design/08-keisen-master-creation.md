## 2. keisen_master.jsonの重要性

### 2.1 プロジェクトへの影響

`keisen_master.json`は、予測システム全体の**基礎データ**であり、以下の理由から極めて重要である：

1. **予測表生成の起点**
   - 予測表生成アルゴリズムの最初のステップで使用される
   - このファイルの内容が予測表の形状を決定する

2. **AI学習データの品質**
   - AIモデルが学習する予測表は、このファイルから生成される
   - ファイルの誤りは、AIモデルの学習品質に直接影響する

3. **再現性の担保**
   - 同じ回号から常に同じ予測表を生成するためには、このファイルが正確である必要がある
   - ファイルの変更は、過去の予測結果にも影響を与える可能性がある

### 2.2 正確性の重要性

`keisen_master.json`の1つの値の誤りが、以下の連鎖的な影響を及ぼす可能性がある：

```
keisen_master.jsonの誤り
  ↓
予測出目の誤り
  ↓
予測表の誤り
  ↓
AI特徴量の誤り
  ↓
AIモデルの精度低下
  ↓
予測結果の信頼性低下
```

そのため、作成時には**慎重な確認と検証が不可欠**である。

## 3. JSON構造の詳細

### 3.1 基本構造

詳細なJSON構造については、[03-data-api-design.md](./03-data-api-design.md#keisen_masterjson)を参照してください。

**構造の要点：**
- トップレベル: `n3` / `n4`
- `columns`: 各桁の定義（百の位、十の位、一の位 または 千の位、百の位、十の位、一の位）
- 階層構造: `桁 → 前回の数字 → 前々回の数字 → 予測出目配列`

### 3.2 データ型

- **桁名**: 文字列（"百の位", "十の位", "一の位" 等）
- **前回の数字**: 文字列（"0" 〜 "9"）
- **前々回の数字**: 文字列（"0" 〜 "9"）
- **予測出目配列**: 数値配列（各要素は0〜9の整数）

### 3.3 必須パターン

以下の全ての組み合わせに対応する必要がある：

- **N3**: 3桁 × 10通り（前回） × 10通り（前々回） = 300パターン
- **N4**: 4桁 × 10通り（前回） × 10通り（前々回） = 400パターン
- **合計**: 700パターン

※ 実際のパターン数は、紙媒体の罫線ルールに依存する可能性がある

## 4. 作成手順の枠組み

### 4.1 準備段階

1. **紙媒体の罫線ルールを入手する**
   - 中国罫線理論のルールブックを確認
   - 各桁、各条件での予測出目ルールを理解

2. **データ構造を理解する**
   - JSON構造を確認（[03-data-api-design.md](./03-data-api-design.md)参照）
   - 前回・前々回の数字から予測出目を導く方法を理解

3. **作成環境を準備する**
   - JSONエディタまたはテキストエディタを準備
   - JSONのバリデーション環境を準備

### 4.2 作成段階

#### 4.2.1 小路データの表構造の理解

小路データの表は以下の構造になっています：

```
┌─────────┬─────────────────────────────┐
│ 前々回   │              0               │ ← 前々回の値（このブロック全体）
├─────────┼───┬───┬───┬───┬─────┤
│ 前回    │ 0 │ 1 │ 2 │ 3 │ ... │ ← 前回の出目（横軸）
├─────────┼───┼───┼───┼───┼─────┤
│         │ 9 │ 1 │ 6 │ 8 │     │
│         │ 8 │ 7 │ 1 │ 0 │     │
│ 予測出目│ 0 │   │ 5 │ 6 │     │ ← 各列に縦に並ぶ
│         │   │   │ 7 │   │     │
│         │   │   │ 9 │   │     │
└─────────┴───┴───┴───┴───┴─────┘
```

**重要なポイント：**
- 1つの表 = 1つの「前々回の値」（0～9）に対応
- 横軸 = 「前回の値」（0～9）
- 各列に縦に並ぶ数字 = その条件での「予測出目」
- 各桁について10個の表（前々回0～9）が存在

**読み取り例：**
- 前々回 = 0、前回 = 2 の場合
- 「前々回 = 0」の表を探す
- 「前回 = 2」の列を探す
- その列に縦に並んでいる数字を上から順に読む：[6, 1, 5, 7, 9]

#### 4.2.2 データ転記の手順

1. **N3用データの作成**
   - 百の位：10表 × 10列 = 100パターンを転記
   - 十の位：10表 × 10列 = 100パターンを転記
   - 一の位：10表 × 10列 = 100パターンを転記

2. **N4用データの作成**
   - 千の位：10表 × 10列 = 100パターンを転記
   - 百の位：10表 × 10列 = 100パターンを転記
   - 十の位：10表 × 10列 = 100パターンを転記
   - 一の位：10表 × 10列 = 100パターンを転記

3. **転記時の注意点**
   - 各セルの数字を**必ず上から下へ**順番に読み取る
   - 空のセルは空配列 `[]` として記録
   - 数字は0～9の範囲であることを確認
   - 転記後、必ず元の表と照合して検証

4. **データの統合**
   - N3/N4のデータを1つのJSONファイルに統合
   - JSON形式の検証

```json
{
  "n4": {
    "千の位": {
      "0": {
```

## 5. 検証方法

### 5.1 JSON形式の検証

- **JSONバリデーション**
  - JSONの構文エラーがないことを確認
  - 必須キーが存在することを確認
  - データ型が正しいことを確認

- **構造の検証**
  - 全ての桁が定義されていることを確認
  - 全ての前回・前々回の組み合わせが定義されていることを確認（0〜9の各組み合わせ）

### 5.2 データの整合性検証

- **予測出目の範囲チェック**
  - 各予測出目が0〜9の範囲内であることを確認
  - 配列が空でないことを確認

- **完全性チェック**
  - 全ての必須パターンが定義されていることを確認
  - 欠損がないことを確認

### 5.3 機能検証

- **予測表生成テスト**
  - 既知の回号で予測表を生成
  - 予測表が正しく生成されることを確認
  - 過去の実績と照合（可能な場合）

- **回帰テスト**
  - 過去に生成した予測表と同じ結果が得られることを確認
  - ファイル更新時の影響範囲を確認

## 6. 注意事項

### 6.1 作成時の注意点

1. **一貫性の維持**
   - 紙媒体のルールを正確に反映する
   - 主観的な解釈を避ける
   - 不明な点は明確にするまで作成を進めない

2. **バージョン管理**
   - 作成したファイルはGitで管理する
   - 変更履歴を明確に記録する
   - 重大な変更は、既存の予測表への影響を確認する

3. **バックアップ**
   - 作成中のファイルは定期的にバックアップする
   - 検証済みのファイルは別途保存する

### 6.2 更新時の注意点

1. **影響範囲の確認**
   - 更新が既存の予測表に与える影響を確認
   - AIモデルの再学習が必要か判断

2. **段階的更新**
   - 一度に全てを更新せず、段階的に更新する
   - 各段階で検証を実施する

3. **ドキュメント更新**
   - 更新内容を本ドキュメントに記録する
   - 変更履歴を明確にする
