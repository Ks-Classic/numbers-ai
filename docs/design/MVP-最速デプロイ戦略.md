#### Q1: 「過去の傾向をつかむ」部分は時間がかかる？
**A: いいえ、データさえあれば自動で学習できます（2-3時間）**

- 手動で傾向を分析する必要はありません
- 100回分のデータで学習すれば、過去の傾向は自動的に反映されます
- 所要時間：データ準備後の学習処理で2-3時間程度

#### Q2: 「番号とリハーサル番号の関係性をつかむ」部分は時間がかかる？

- MVPでは、以下の簡易的な特徴量で十分：

  - リハーサル数字の位置（重心座標）
  - リハーサル数字との距離（簡単な距離計算）
- 詳細な関係性分析は、Phase 3以降で実施します
- 所要時間：1-2日で実装可能

## 🚀 MVP最速デプロイ戦略（7日間）

### Day 0（準備日）: keisen_master.json作成の効率化

**重要**: 開発開始前に、keisen_master.jsonを作成しておくことを推奨します。

**効率化方法:**
1. **OCRツールを活用**（`scripts/extract_from_image.py`）
   - 画像から自動でJSON生成
   - PDF補完機能も利用可能
   - 手作業の70%を自動化可能

2. **分割作業**
   - N3/N4を分けて作業
   - 1桁ずつ（100パターン）を作成して検証
   - 完了したら統合（`scripts/merge_keisen_files.py`）

3. **並行作業**
   - 画像準備とOCR処理を並行
   - 検証は後回しでもOK（後で修正可能）

**所要時間**: OCRツール活用で1-2日に短縮可能（手作業だと3-4日）

### Day 3-4: AIモデル構築（最速化）

**タスク:**
- ✅ データ前処理（Jupyter Notebook）
- ✅ 特徴量エンジニアリング（MVP簡易版）
- ✅ XGBoostモデル学習（100回分データ）
- ✅ モデル評価（基本的な指標のみ）

**MVP簡易版特徴量（時間短縮のため）:**

```python
# 簡易版特徴量（MVP用）
# 詳細な関係性分析はPhase 3以降で実施

# 1. 形状特徴（基本的なもののみ）
- 各数字の出現頻度
- 各数字の最大連続長

# 2. 位置特徴（基本的なもののみ）
- 各数字の重心座標
- 各数字の端からの距離

# 3. リハーサル関係性（簡易版）
- リハーサル数字の出現回数
- リハーサル数字の位置（重心）
- リハーサル数字との距離（ユークリッド距離）

# 4. 集約特徴（基本的なもののみ）
- 組み合わせの出現頻度
- 分散度
```

**時間短縮のコツ:**
- 特徴量は10-20個程度の簡易版でOK（Phase 3で拡張）
- ハイパーパラメータはデフォルト値でOK（Phase 3で最適化）
- モデル評価は基本的な指標のみ（AUC-ROC, Precision, Recall）

**所要時間**: 2-3時間（データ準備後の学習処理）
