## ✅ 結論：アーカイブは不要

### 理由

1. **最新のデータファイル**
   - 全てのデータファイルは最新のLightGBMモデル学習（2025-01-18）に使用されたもの
   - 「古いデータ」ではなく「最新のデータ」

2. **再生成に時間がかかる**
   - 組み合わせ予測データの生成には数時間かかる
   - 再生成する必要がある場合、時間とリソースを消費する

3. **推論時には不要だが、学習時には必要**
   - 推論時には不要（`.vercelignore`で除外済み）
   - 新しいモデルを学習する際には必要

4. **ディスク容量の問題**
   - 合計約6.1GBと大きいが、再生成コストを考えると保持する価値がある

## 📦 アーカイブが必要な場合

### 条件

以下の場合のみ、アーカイブを検討：

1. **ディスク容量が不足している**
2. **新しいデータファイルを生成する予定がある**
3. **バックアップとして保持したい**

### アーカイブ方法

```bash
# バックアップディレクトリに移動
mkdir -p data/models/backup/lightgbm_data_2025-01-18

# データファイルをバックアップに移動
mv data/models/*_data.pkl data/models/backup/lightgbm_data_2025-01-18/

# または、シンボリックリンクを作成（ディスク容量を節約）
ln -s data/models/backup/lightgbm_data_2025-01-18/*_data.pkl data/models/
```

## 🔄 再生成が必要な場合

### 新しいモデルを学習する際

1. **特徴量を追加・変更した場合**
   - データファイルを再生成する必要がある
   - `notebooks/run_03_feature_engineering_full.py`を実行

2. **データ範囲を変更した場合**
   - データファイルを再生成する必要がある
   - `notebooks/run_03_feature_engineering_full.py`を実行

3. **既存のデータファイルを使用する場合**
   - 現在のデータファイルを使用可能
   - 再生成は不要
